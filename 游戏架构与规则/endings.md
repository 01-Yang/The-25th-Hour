# 结局系统

## 结局入口

结算后统一进入 `结局与成就`。

其中只有两个页签：

- 人生结局
- 成长成就

## 结局结构

结局只分为两大类：

- 成功通关结局：完整读完五年
- 中途失败结局：在五年中途被系统判定出局

成功通关结局采用：

- 1 个主结局
- 1 句状态收尾

不做正式多层组合结局，也不再区分“硬失败 / 软失败”。

状态很差但仍然读完五年的情况，仍然算作成功通关，只是在对应主结局下给出更沉重的状态收尾。

## 成功通关结局池 V1

| 结局名 | 核心触发方向 |
|---|---|
| 顶级事务所 | 高设计水平，高作品集，秋招进入强事务所 |
| 普通事务所 | 事务所路线达成，但综合表现未进顶级档 |
| 保研直升 | 保研资格达成并成功录取 |
| 考研上岸 | 考研路线成功 |
| 海外留学 | 留学申请成功 |
| 体制内上岸 | 考公或考编路线成功 |
| 选调回乡 | 选调路线成功 |
| AI 新职业 | AI 路线成功，软件技术突出 |
| 创业起步 | 创业路线成功，有较强资源和履历支撑 |
| 转行离场 | 秋招或毕业阶段主动转行成功 |
| 普通但稳定 | 没有冲进强路线，但顺利毕业且状态平稳 |

## 中途失败结局池 V1

| 结局名 | 触发方向 |
|---|---|
| 生活费断裂 | 金钱长期为负且未能及时回正 |
| 压力失控 | 压力持续爆表并触发崩盘 |
| 连续挂科退学 | 连续两次 F 后触发退学 |
| 被迫停学 | 综合状态极差，无法继续维持学业 |

## 结局读取维度

- 最终主路线
- GPA
- 设计水平
- 软件技术
- 作品集
- 实习 / 工作经历
- 竞赛结果
- 精力与压力形成的长期状态

## 状态收尾

状态收尾不单独构成结局类型，只作为主结局后的最后一句话。

首版建议按三档收尾：

- 昂扬：理想尚在，状态较好
- 复杂：目标达成，但内里已有代价
- 沉重：虽然通关，但人已经被五年生活明显改变

## 成长成就

- 成长成就是总库
- 不做分类树
- 只要达成条件就弹出提示并永久收录
