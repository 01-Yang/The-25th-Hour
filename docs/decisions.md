# 决策记录

本文只保留对后续开发仍有解释价值的关键决策，不作为现行规则源。现行规则以 `PRD.md -> systems.md -> numbers.md -> 内容文档` 为准。

## ADR 是什么

ADR 是 Architecture Decision Record，即“架构决策记录”。它记录的是：当时为什么在几个方案里选择了某个结构性做法。ADR 不重复规则全文，不替代 `systems.md` 或 `numbers.md`。

## 当前关键决策

### 随机事件体系

- 事件文档按 `普通随机事件 / 交互事件 / 模型周事件 / 万里路事件` 维护。
- AI 相关内容不单独拆主系统，只作为事件语境、AI 相关经历和后期路线条件。
- 事件开发配置统一使用结构化字段：`event_id / title / pool / phase / sentiment / tags / context_tags / base_weight / cooldown_weeks / repeatable / ai_related / ai_experience_delta / trigger_rules / result / options / note`。
- 首版可以先消费最小字段完成抽取与结算，但字段口径不再拆第二套。

### 竞赛结果

- 竞赛每学期最多投稿 `1` 次，同一课题作品只能投稿 `1` 次。
- 图纸等级只做投稿门槛；是否获奖读取作品分概率表。
- 奖项只保留 `未获奖 / 三等奖 / 二等奖 / 一等奖`。
- 竞赛奖金、简历记录和作品集分开结算；竞赛不直接增加作品集。
- 玩家可见层只展示奖项结果和 `竞赛奖金 ￥{amount}`，不再单独弹“简历写入”或“竞赛成就”。

### 实习价值

- 建筑工作路线读取累计 `实习价值`：普通实习 `+1`，强事务所实习 `+2`，名企实习 `+3`。
- 实习申请进阶读取最高已完成实习档位，不能用累计值替代。
- 这样 `普通 + 强事务所 = 3` 可以满足建筑工作路线履历厚度，但不会被误判为已经完成名企实习。
- 实习玩家可见层收口为录取、拒绝和三周短事件；入职、结束总结和简历写入不再作为独立弹窗。

### 开发冻结

- 首版不再新增主系统，只推进实现落地、内容补全、数值调参和体验打磨。
- `GameState` 是唯一真相源；时间推进走显式状态机；所有数值变化走统一结算器。
- 固定 seed 必须能复现事件抽取、考试抽题、实习录取、留学申请和岗位投递内部判定。

### 模拟结论

- 当前 `simulator` 是首个可执行规则内核，不是第二套规则源。
- 已验证的模拟结论只用于支持调参和实现迁移；稳定口径已并回 `systems.md`、`numbers.md`、`content-plan.md`、`events.md`、`endings.md` 和本文。
- 后续网页实现应逐步从 `simulator` 抽出正式 `game-core`；纯研究型报告入口已删除。

### 调参追溯

- 已确认的调参原则已并入 `numbers.md` 的对应章节。
- 若后续可玩版本出现强路线整体过易，优先按 `numbers.md` 的“强路线稀有度”调整路线门槛。
