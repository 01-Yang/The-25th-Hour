# 结局系统

## 结局入口

结算后统一进入 `结局与成就`。

页签固定为：

- 人生结局
- 成长成就

## 结局结构

结局只分为两类：

- 成功通关结局：完整读完五年
- 中途失败结局：在五年中途被系统判定出局

成功通关结局统一采用：

- `1` 个主结局
- `1` 句状态收尾

不做多层组合结局，也不再区分“硬失败 / 软失败”。

## 成功通关主结局

### 路线组

- 升学路线组
- 体制内路线组
- 建筑工作路线组
- 转行路线组

### 具体结局池

| 结局名 | 触发方向 |
|---|---|
| 保研直升 | 升学路线组中保研成功 |
| 考研上岸 | 升学路线组中考研成功 |
| 海外留学 | 升学路线组中留学成功 |
| 体制内上岸 | 体制内路线组中考公或考编成功 |
| 选调回乡 | 体制内路线组中选调成功 |
| 顶级事务所 | 建筑工作路线组中进入外企事务所或大师事务所高档 |
| 普通事务所 | 建筑工作路线组达成，但未进入顶级档 |
| AI 新职业 | 转行路线组中 AI 产品经理成功 |
| 转行离场 | 转行路线组中非 AI 岗位成功 |
| 普通但稳定 | 未进入强路线，但顺利毕业且状态稳定 |

### 成功结局读取优先级

当多个成功路线同时满足时，只读取 `1` 个主结局。

统一优先级如下：

1. 升学路线组
2. 体制内路线组
3. 建筑工作路线组
4. 转行路线组
5. 普通但稳定

同一路线组内再按下列顺序读取：

- 升学路线组：`保研直升 > 海外留学 > 考研上岸`
- 体制内路线组：`选调回乡 > 体制内上岸`
- 建筑工作路线组：`顶级事务所 > 普通事务所`
- 转行路线组：`AI 新职业 > 转行离场`

读取规则：

- 强路线只影响主结局优先级，不构成失败条件
- 某条强路线不满足时，继续读取同组下一条或下一路线组
- `普通但稳定` 是完整读完五年且未中途失败的最终保底
- 如果多个体制内方向同时满足，优先读取 `选调回乡`

### 成功通关保底

- 只要玩家未触发中途失败，并完整读完五年，就一定进入成功通关结算。
- 如果没有达成任何强路线，则进入 `普通但稳定`。
- `普通但稳定` 不要求作品集、竞赛、实习或 AI 相关经历达标。

## 中途失败结局

| 结局名 | 触发条件 |
|---|---|
| 生活费断裂 | 任意结算后金钱归零 |
| 压力失控 | 压力 `> 90` 连续 `2` 周 |
| 连续挂科退学 | 连续 `2` 次评图为 `F` |
| 被迫停学 | 同时满足 `精力 = 0` 且压力 `> 90`，并在周末结算时仍未脱离 |

### 失败判定规则

- 中途失败统一在周末结算、评图结算或学期结算节点触发。
- 只有 `生活费断裂` 例外：任意结算后金钱归零立即失败。
- 大一第一学期不触发 `压力失控` 与 `被迫停学`。
- 大一第一学期仍可能触发 `生活费断裂` 与 `连续挂科退学` 的前置警告，但不会直接因单次 `F` 出局。

### 失败警告

- 单学期出现 `1` 次 `F`：给出退学警告
- 压力首次超过 `90`：给出崩盘警告
- 精力降到 `0`：给出强警告，并立即进入高危状态

## 状态收尾

状态收尾不单独构成结局类型，只作为主结局后的最后一句话。

三档固定为：

- 昂扬：理想尚在，状态较好
- 复杂：目标达成，但内里已有代价
- 沉重：虽然通关，但人已经被五年生活明显改变

## 结局读取维度

- 最终主路线
- GPA
- 设计水平
- 软件技术
- 作品集
- 实习 / 工作经历
- 竞赛结果
- 精力与压力形成的长期状态

## 成长成就

- 成长成就是总库
- 不做分类树
- 首版总量固定为 `30`
- 达成条件后立即弹出并永久收录
