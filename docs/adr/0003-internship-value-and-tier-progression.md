# 实习价值累计与实习进阶拆分

在“第二十五小时”当前版本里，实习系统拆成两个读取口径：建筑工作路线读取累计 `实习价值`，实习申请进阶读取最高已完成实习档位。

累计 `实习价值` 表示玩家已经完成的实习履历厚度：每完成 1 段普通实习记 `+1`，每完成 1 段强事务所实习记 `+2`，每完成 1 段名企实习记 `+3`。建筑工作路线只按这个累计值判断基础履历厚度，因此 `普通 + 强事务所 = 3` 可以满足读取 `实习价值 >= 3` 的路线门槛。

实习申请进阶不能读取累计值，而必须读取最高已完成档位。原因是累计 `普通 + 强事务所 = 3` 代表履历厚度，不代表玩家已经完成过名企实习；如果把累计值误作最高档位，系统会错误地跳过名企申请。这个拆分让路线读取承认多段实习的积累，也让实习申请保持清晰的档位递进。

玩家可见层同步收口为：申请成功只显示 `你收到了一封实习录取邮件。`，申请失败只显示 `你盯着那封拒绝信看了很久。`，三周实习过程由短事件提供连续体感。入职提示、结束总结和简历写入提示不再作为独立弹窗存在，相关信息只进入结构化记录、个人简历和路线读取。
