# 游戏引导、外部链接与支持页面

本文确认游戏引导、BGM 设置、飞书外链入口和“请作者喝咖啡”页面。

## BGM 与声音设置

- 音乐系统首版只做表现层，不提供数值加成。
- 开头曲和中间 BGM 不显示歌词。
- 结束曲显示歌曲介绍弹窗、底部歌词字幕和专辑旋转。
- 设置项保存到 `localStorage`：主音量、音乐音量、音效音量、静音、动效减弱。
- 玩家切换学年 BGM 时，只能在当前学年 `3` 首歌内切换。
- 结束曲播放期间不能切歌。
- 页面失焦时首版只暂停动效，不自动改音量。

## 游戏引导

首版引导目标是让玩家理解“每周 3 次行动”和“进度/质量/状态取舍”，不把玩家推向某条最优路线。

推荐引导结构：

1. 开局档案：解释昵称和大学名称只用于显示，不提供数值加成。
2. 角色选择：解释角色有被动特质、专属技能和初始面板。
3. 第一周行动：提示每周固定 `3` 次行动。
4. 第一次评图前：提示进度回答“有没有做完”，作品质量回答“做得好不好”。
5. 第一次高危状态：提示精力、压力、金钱风险和行动置灰原因。
6. 大五路线入口：不主动弹窗催促，只在玩家打开路线页时展示入口说明。
7. 毕业典礼：不解释规则，只做尾声表现。

引导形式：

- 首次进入关键系统时使用短提示，不连续轰炸。
- 所有提示可跳过。
- 设置页提供“重看游戏引导”入口。
- 规则说明不要写成攻略答案，不展示隐藏成功率、路线考试分数线内部折算或结局读取优先级细节。

## 飞书外链按钮

公告系统、社区系统和作者的话均对接飞书链接。链接在配置中维护，未填写时入口置灰并提示“链接待填写”。

| 入口 | 配置键 | 默认值 | 点击行为 |
|---|---|---|---|
| 公告系统 | `externalLinks.announcementFeishuUrl` | 待填写 | 新标签页打开飞书链接 |
| 社区系统 | `externalLinks.communityFeishuUrl` | 待填写 | 新标签页打开飞书链接 |
| 作者的话 | `externalLinks.authorNoteFeishuUrl` | 待填写 | 新标签页打开飞书链接 |

点击规则：

- 已填写链接：`window.open(url, "_blank", "noopener,noreferrer")`。
- 未填写链接：按钮置灰，不跳转。
- URL 必须由配置提供，不硬编码在组件里。
- 飞书链接变化不影响存档和规则。

## 请作者喝咖啡

“请作者喝咖啡”按钮打开游戏内支持页面，不直接跳转外站。

页面建议路径：`/support-coffee`

页面内容：

- 标题：`请作者喝咖啡`
- 简短文字：`如果这五年的故事陪你坐了一会儿，也欢迎请作者喝杯咖啡。`
- 微信收款码图片：`assets/support/wechat-qr.png`
- 支付宝收款码图片：`assets/support/alipay-qr.png`
- 图片缺失时显示占位状态：`收款码待上传`
- 页面底部保留返回按钮。

交互规则：

- 两张二维码并排展示；移动端上下排列。
- 图片只作为静态资源展示，不上传、不识别、不记录支付状态。
- 不做打赏排行榜，不做支付成功回调。
- 支持页面不进入游戏规则状态，不写入 `GameState`。
