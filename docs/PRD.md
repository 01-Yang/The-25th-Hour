# 第二十五小时：建筑生模拟器 PRD

## 一句话定义

一款让玩家重新经历五年建筑学院青春的人生模拟器。

## 核心目标

这不是纯粹的建筑设计训练模拟。

它真正要模拟的是：

- 理想和现实之间的拉扯
- 建院生活中的时间分配
- 评图、赶图、生活费、导师、AI 冲击带来的持续压力
- 玩家如何在五年里慢慢长成一种人

## 产品方向

本项目的首要原则是：

- 玩法简洁
- 结构清楚
- 不再继续扩主系统
- 所有规则都要能直接落地实现

首版定位为“建筑生人生模拟器”，不是纯建筑设计训练模拟器。建筑专业负责承载压力、成长和人生分岔；玩家真正管理的是时间、状态、金钱、课题和后期方向。

## 核心结构

- `5` 年制，共 `60` 周
- `1` 学年 = `12` 周
- `1` 学期 = `6` 周
- 每周固定 `3` 次行动
- 第 `5` 周为模型周
- 第 `6` 周为赶图周
- 赶图周结束后直接进入评图阶段

## 开局流程

1. 随机抽取 `3` 张角色卡
2. 可免费重抽 `1` 次
3. 选择 `1` 个角色
4. 进入开学典礼、军训和建院生活开始提示
5. 军训结束后，从 `3` 位导师中选择 `1` 位
6. 正式进入第一学期

## 玩家成长结构

### 长期值

- 六项角色属性
- 精力
- 压力
- 金钱
- 个人 GPA
- 作品集
- 个人简历
- AI 相关经历

### 学期临时值

每学期只保留两个课题临时值：

- 进度
- 作品质量

学期结束后清零；大五上和大五下作为同一个毕业设计例外，进度与作品质量连续继承，且大五毕业设计的进度与作品质量上限均为 `200`。

## 核心循环

### 每周循环

1. 进入新的一周
2. 系统按阶段规则触发本周事件
3. 玩家执行 `3` 次行动
4. 周末统一结算行动、事件、商店和实习后果
5. 执行失败检查、每周自动恢复和高危限制更新
6. 推进到下一周

### 每学期循环

1. 第 `1-4` 周：正常推进
2. 第 `5` 周：模型周
3. 第 `6` 周：赶图周
4. 评图阶段
5. 学期结算

## 角色系统

- 每个角色拥有：
  - 初始精力
  - 初始压力
  - 初始金钱
  - 六项角色属性
  - `1` 个被动特质
  - `1` 个专属技能
- 专属技能统一占用 `1` 次行动机会
- 专属技能统一冷却 `10` 周
- 角色差异主要来自初始面板、被动特质和专属技能
- 不再新增专属资源或额外行动槽

## 导师系统

- 军训结束后进行第一次导师选择，之后每个新学年重新选择导师
- 每次随机 `3` 选 `1`
- 不设置导师认可度
- 每位导师每学期发布 `1` 个阶段任务
- 阶段任务只统计当学期行为
- 新学期开始时阶段任务进度重置
- 阶段任务必须绑定已有行动或已有系统
- 阶段任务统一在评图前结算

导师影响固定为：

- 阶段任务
- 事件倾向
- 路线倾向

## 评图系统

- 评图是每学期最核心的结算
- 评图采用“进度硬门槛 + 基础图纸评级 + 汇报策略修正最终评级”
- 大一至大四评图前进度必须达到 `100 / 100`，否则本次评图直接为 `F`
- 大五上期中汇报前毕业设计进度必须达到 `100 / 200`，否则本次评图直接为 `F`
- 作品质量先折算为质量评图分，再生成基础图纸评级
- 质量评图分未达到 `F` 门槛时，基础图纸评级为 `F`
- 汇报策略不修正进度或作品质量，只在基础图纸评级后影响最终图纸评级、作品分和设计课绩点
- 进度未达硬门槛导致的 `F` 不可补救；质量不足导致的 `F` 可以用 `求求你别挂我` 低概率补救为 `D`
- 评图等级固定为 `S / A / B / C / D / F`
- `D` 视为过关
- `F` 视为挂科

## 大学课程

- 每学期开始选择 `1` 门大学课程
- 大学课程不做独立玩法
- 大学课程不占行动次数
- 大学课程只承担：
  - 属性倾向
  - 事件倾向
  - GPA 修正
- 学期结束答 `3` 道题，直接得到课程修正

## 副系统边界

### 建筑生的万里路

- 大二上开始在满足条件时可用
- 每次消耗 `2` 次行动机会和 `2500` 金钱
- 可在模型周或赶图周使用
- 万里路事件必定触发，且不计入普通周最多 `2` 次事件上限
- 属于成长副系统，不改变主课题结构
- 不进入作品集，只进入简历与成长路径

### 暑假写生

- 固定 `2` 个节点
- 只提供属性与事件收益
- 不单独设置失败条件

### AI 相关语境

- 从大一下开始持续进入事件与路线
- 不拆分成独立主系统
- 不保留 AI 立场设定，只通过 AI 相关经历和后期路线体现

## 实习与后期路线

- 实习从大一开始开放
- 实习入口未满足最低门槛时仍展示，并提示触发条件
- 每学期最多申请 `1` 次
- 实习申请不消耗行动次数，并采用概率录取
- 同学期申请失败后不能重投
- 实习录取后一般持续 `3` 周，跨学期时继承剩余周数
- 已完成实习的单位不能再次申请
- 实习结束时展示实习总结

后期路线按四个路线组收束：

- 升学路线组
- 体制内路线组
- 建筑工作路线组
- 转行路线组

后期路线从大四上第 `1` 周开始提醒玩家准备选择，但不强制选择。玩家可以先设置可修改的路线意向；只有大五阶段完成路线报名、申请、投递、考试或确认后，才形成路线正式参与记录。未正式参与方向时仍可继续周行动、实习、竞赛投稿、商店和常规推进。每局最多正式参与 `保研 / 考研 / 留学 / 选调 / 考公 / 考编 / 建筑工作 / 转行` 之一；选择建筑工作时必须同步确认 `1` 个投递岗位，选择转行时必须同步选择具体岗位。若毕业前仍未正式参与方向，结局系统跳过路线结果，读取普通状态毕业结局；若已正式参与方向，大五阶段只按已参与方向做最终判定，不跨路线补读结局。

## 经济系统

- 生活费按家庭背景固定发放
- 金钱为强约束资源
- 余额不足时，带花费行为不可执行
- 任意结算后金钱小于 `0`，直接失败；金钱等于 `0` 不触发失败
- 商店只做轻量修正，不做 build 系统

## 失败与通关

- 只有完整读完五年、未中途失败且完成毕业设计，才算成功通关
- 完整读完五年但未完成毕业设计时，读取 `毕业失败——延毕`
- 中途失败采用少量明确条件，不做复杂连锁出局树
- 成功通关只读取 `1` 个主结局 + `1` 句状态收尾

结局读取优先级固定为：中途失败 -> 毕业失败——延毕 -> 已正式参与路线结果 -> 状态毕业。

## 开发落地原则

- 游戏状态使用统一 `GameState`，UI 只读取状态并发起操作。
- 时间推进使用显式状态机，不由单个页面自行决定流程。
- 所有数值变化走统一结算器，并产出结构化变化和结算日志。
- 行动入口由统一可用性解析器返回可用、置灰或隐藏状态。
- 每局使用固定随机种子，保证事件抽取、考试抽题和概率结果可复现。
