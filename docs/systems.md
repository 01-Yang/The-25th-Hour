# 游戏系统架构

## 总体结构

游戏按 `周 -> 学期 -> 学年 -> 五年` 推进。

- 一学期：`6` 周
- 一学年：`12` 周
- 完整流程：`5` 年，共 `60` 周
- 每周固定：`3` 次行动
- 第 `5` 周：模型周
- 第 `6` 周：赶图周
- 赶图周结束后：进入评图阶段

首版体验定位：

- 游戏是“建筑生人生模拟器”，不是纯建筑设计训练模拟器。
- 建筑专业负责承载时间分配、压力、成长和结局，不追求完整复刻真实教学流程。
- 每周核心压力来自固定 `3` 次行动机会的取舍，不来自复杂菜单、隐藏系统或多路线并行管理。
- 普通通关应稳定可达，强路线保持稀有，中途失败要有明确预警。

### 程序状态机

开发实现必须以显式阶段状态机推进，不允许由 UI 页面各自决定流程。

推荐阶段顺序：

1. 开局档案填写
2. 开局角色选择
3. 固定流程事件
4. 学期开始
5. 周开始
6. 周行动
7. 周结算
8. 评图阶段
9. 学期结算
10. 年度切换
11. 后期路线提醒与正式参与
12. 毕业典礼
13. 结局结算

规则：

- 游戏状态必须写入统一 `GameState`，UI 只读状态并发起操作。
- 行动、事件、商店、实习、评图、路线和结局都只能通过统一结算器修改状态。
- 每次结算都必须产出结构化数值变化和结算日志。
- 每局生成固定随机种子；随机事件、考试抽题、实习录取、留学申请和岗位投递内部判定都必须可复现。
- 自动存档点固定为：每周开始、每周结束、评图前、评图后、路线提醒后、路线正式参与后、结局结算前。

## 角色系统

- 游戏刚开始先填写玩家昵称和大学名称，写入统一 `GameState`。
- 玩家昵称和大学名称只用于称呼、存档、固定剧情、个性化文本和排行榜展示，不提供数值加成，不参与结局门槛。
- 完成开局档案填写后，才进入角色抽取与选择流程。
- 开局随机 `3` 张角色卡
- 玩家可免费重抽 `1` 次
- 最终选择 `1` 张角色卡开始游戏
- 角色额外记录学历背景，用于塑造开局差异与部分路线语境
- 每个角色固定拥有：
  - 初始精力
  - 初始压力
  - 初始金钱
  - 六项角色属性
  - `1` 个被动特质
  - `1` 个专属技能

规则收束：

- 所有角色初始精力固定为 `100`
- 不新增额外行动槽
- 不新增角色专属资源
- 被动特质尽量只修改既有数值规则
- 专属技能统一占用 `1` 次行动机会，统一冷却 `10` 周

## 导师系统

- 军训结束后进行第一次导师选择
- 之后每个新学年重新选择导师
- 每次从 `3` 位导师中选择 `1` 位
- 不设置导师认可度
- 每位导师每学期给 `1` 个阶段任务
- 阶段任务只统计当学期行为、当学期课题状态和本学期属性成长量
- 新学期开始时阶段任务进度重置，不继承上学期的行动计数或任务完成状态
- 阶段任务统一在评图前结算
- 阶段任务只做“完成 / 未完成”判定，不做半完成档
- 阶段任务文本在学期第 `1` 周完整公开

导师影响固定为：

- 阶段任务
- 事件语境

### 阶段任务开放原则

- 早期学期不出现依赖高阶系统的任务
- 依赖 `竞赛投稿 / 名企实习` 等高阶系统的任务，只在对应系统已经开放后出现
- 不能要求玩家在“评图后”才能完成当学期任务
- 不能依赖当学期最终评图等级；如需表现“稳定通过”，只读取评图前可判定的过程指标或已完成学期记录

## 课题系统

每学期课题只保留两个临时值：

- 进度：`0-100`
- 作品质量：`0-100`

学期结束后清零。

例外：

- 大五上和大五下视为同一个 **毕业设计**，进度与作品质量不在大五上结束后清零。
- 大五毕业设计的进度与作品质量上限均为 `200`。
- 大五上为毕业设计期中汇报，期中汇报前毕业设计进度必须达到 `100`，作品质量至少达到不被评为 `F` 的门槛。

进度主要来自：

- 正常速度画图
- 爆肝通宵赶图
- 部分随机事件
- 部分交互事件
- 部分专属技能

作品质量主要来自：

- 方案推敲
- 场地调研
- 模型周材料选择
- 模型周事件
- 部分随机事件
- 部分交互事件
- 部分专属技能

联动原则：

- `正常速度画图` 与 `爆肝通宵赶图` 主负责 `进度`；所有带 `基础进度` 的常驻行动都读取 `软件技术` 档位修正。
- 普通学期中，约 `6-7` 次正常速度画图应能补满进度；约 `3-4` 次爆肝通宵赶图应能补满进度，但状态代价显著更高。
- 事件、导师任务和角色技能允许造成进度加减，因此实际画图次数可以围绕目标压力浮动。
- `方案推敲` 与 `场地调研` 主负责 `作品质量`，并固定附带 `进度 +1`。
- 所有带 `基础作品质量` 的常驻行动都读取 `设计水平 + 创意审美` 的简单档位修正。
- `方案推敲` 更偏室内高压深化，消耗精力并增加压力；`场地调研` 不设每周次数上限，更耗精力和金钱，但略微降低压力。
- 进度与质量合计不应吃掉整个学期；普通过关路线应能留下行动给学习、运动、放松、社交、兼职和突发补救。
- 画图速度与质量成长拆开处理，避免“快”与“好”在同一行动里同时滚满。

## 周行动系统

### 固定行动数

- 每周固定 `3` 次行动
- 不因角色、导师、被动、技能、副系统增加或减少
- 唯一例外为商店道具 `星巴克会员周卡`：购买后仅使`下一周`额外获得 `1` 次行动，总计 `4` 次，且同一周内此效果不叠加

### 不占行动次数的系统

- 选课
- 商店购物
- 竞赛投稿判定
- 实习申请判定
- 学期末课程题
- 年度雅思考试

### 开局固定流程

- 填写玩家昵称和大学名称、选择角色并开始游戏后，先进入 `开学典礼` 固定事件。
- 开学典礼结束后进入 `军训` 固定事件。
- 开学典礼与军训期间，所有周行动、商店、竞赛、实习、路线报名入口均不可使用。
- 军训结束后显示 `建院生活开始提示`。
- 建院生活开始提示结束后，玩家从 `3` 位导师中选择 `1` 位作为大一导师。
- 导师选择完成后，正式进入大一上第 `1` 周。

### 尾声固定流程

- 大五下毕业设计答辩完成后，若未触发中途失败，系统先完成结束曲抽取，再进入 `毕业典礼` 固定事件。
- `毕业典礼` 开始第一帧，已抽取完成的结束曲开始播放；玩家不能切换。
- 毕业典礼事件结束后，触发统一结局系统。
- 结尾动画方案当前不参与游戏系统架构口径，后续单独维护。
- 毕业典礼不再结算属性、金钱、课题或路线结果，只承担尾声表现与情绪收束。

### 大四路线提醒

- 大四上第 `1` 周进入时弹出路线提醒 UI。
- 提醒文案读取 `content-plan.md` 的“后期路线提醒文案”。
- 该 UI 只提醒玩家规划后期路线，不强制选择，不产生路线正式参与记录。
- 玩家在大四阶段选择的路线只作为“路线意向”；路线意向可改。

### 大五路线提醒

- 大五上第 `1` 周进入时弹出路线开放 UI。
- 提醒文案读取 `content-plan.md` 的“后期路线提醒文案”。
- 该 UI 仍不强制选择；只有后续路线报名、申请、投递、考试或确认才会形成路线正式参与记录。

### 行动次数限制

- `接设计外包` 达到至少 `1` 个外包项目门槛后即可选择，每周最多 `2` 次；每次都正常占用 `1` 次周行动，并在项目表中选择具体外包项目结算。
- `校外兼职` 无能力门槛，每周最多 `2` 次；每次都正常占用 `1` 次周行动，并在兼职项目表中选择具体项目结算。
- 高危状态仍可禁用 `接设计外包 / 校外兼职`，高危限制优先级高于每周次数上限。

### 行动可用性

所有行动入口统一由可用性解析器判断。

解析结果只允许三类：

- `可用`：玩家可以直接选择。
- `置灰`：入口可见，但因资源、阶段、高危状态、路线正式参与互斥或次数限制暂时不可用，并显示原因。
- `隐藏`：当前阶段不应展示该入口，例如未开放路线或已关闭的万里路入口。

UI 不直接复制条件逻辑，只展示解析器返回的状态和原因文案。

### 汇报策略

- 汇报策略只在评图阶段选择
- 不提前展开为独立主玩法
- 汇报策略不再修正 `进度` 或 `作品质量`
- 汇报策略在基础图纸评级产生后结算，只影响最终图纸评级、作品分和设计课绩点
- 汇报成功率由策略固定值决定，属性只作为可选门槛
- 汇报策略只读取策略自身门槛和基础图纸评级；导师不对汇报策略提供额外加减分。
- 进度未达硬门槛导致的 `F` 不展示可补救策略，直接进入挂科结果。
- 质量不足导致的基础图纸评级 `F` 仍可选择 `求求你别挂我` 低概率补救为 `D`，其余策略不能救 `F`。
- 汇报策略完成后，UI 先显示汇报成功或失败，再展示最终图纸评级与作品分。

### 高危状态行动限制

当进入高危状态时，行动界面必须提示玩家当前风险，并限制部分行动选项。

- 精力高危：`精力 < 30`
- 压力高危：`压力 > 80`
- 精力高危时，禁用 `爆肝通宵赶图 / 接设计外包 / 校外兼职 / 万里路 / 实习申请` 等高精力消耗或持续消耗入口。
- 压力高危时，禁用会增加压力的常驻行动、专属技能和交互事件选项。
- 高危状态下至少保留 `休养生息` 和当前可购买且不会增加对应高危资源风险的商店恢复商品，避免玩家无行动可选。
- 高危限制只限制行动选择，不直接触发结局；结局仍按失败阈值在结算节点读取。
- 轻度风险时，行动产生的正向 `进度 / 作品质量 / 六项属性` 各自 `-1`，最低仍保留 `+1`。
- 高危时，行动产生的正向 `进度 / 作品质量 / 六项属性` 各自 `-2`，最低仍保留 `+1`。
- 若玩家选择的行动或事件选项在结算后导致精力小于 `0`，不额外弹出风险确认，直接触发 `被迫停学`。
- 首次进入状态高危时弹出提示：`你正在崩溃边缘，别画图了，快去休息！`
- 若当前金钱低于该角色下周自动生活费扣费额，触发金钱高危提示：`你快要饿死了，省着点花！`
- 金钱高危只做预警和花费行为置灰，不直接触发失败；金钱小于 `0` 时才读取 `生活费断裂`。

## 评图系统

评图阶段不再触发普通随机事件。

评图采用“进度硬门槛 + 基础图纸评级 + 汇报策略修正最终评级”的三段制。

进度硬门槛：

- 大一至大四每学期课题进度上限为 `100`；评图前进度未达到 `100` 时，本次评图直接为 `F`，不可用汇报策略补救。
- 大五上为毕业设计期中汇报；毕业设计进度未达到 `100` 时，本次评图直接为 `F`，不可用汇报策略补救。
- 大五下为毕业设计答辩；毕业设计最终进度未达到毕业门槛时，视为未完成毕业设计。

质量硬门槛：

- 大一至大四作品质量未达到 `F` 门槛时，基础图纸评级为 `F`。
- 大五毕业设计作品质量按 `0-200` 记录，评图时折算为 `0-100` 的质量评图分；折算后未达到 `F` 门槛时，基础图纸评级为 `F`。
- 基础图纸评级先由作品质量评图分决定。
- 汇报策略不改变作品质量，只在基础图纸评级之后改变最终图纸评级、作品分和设计课绩点。
- 导师阶段任务和部分事件只允许在评图前修正作品质量、状态或 GPA，不再产生独立“评图总分”。

评图等级固定为：

- S
- A
- B
- C
- D
- F

规则：

- `D` 视为过关
- `F` 视为挂科
- 每次评图结束后，系统必须告诉玩家本次最终图纸评级是 `S / A / B / C / D / F` 中的哪一档，并显示本次作品分。
- 连续 `2` 次 `F` 直接触发 `连续挂科退学`

## GPA 系统

- GPA 范围：`0.0-4.0`
- 开局显示 GPA：`4.0`
- GPA 由学期评图产生的设计课绩点叠加本学期 `GPA学期修正` 构成
- 大学课程是最稳定的 `GPA学期修正` 来源；少数导师阶段任务和角色技能也可以写入该修正
- GPA 服务于后期路线，不反向改变每周行动结构

## 大学课程系统

- 每学期开始可从课程目录中选择 `1` 门课
- 课程选择不占每周行动次数
- 大学课程不做独立玩法
- 大学课程的作用固定为：
  - 属性收益
  - 事件语境
  - GPA 学期修正

学期结束：

- 回答 `3` 道对应课程题
- 直接结算课程修正
- 答题过程中不显示单题对错，也不显示正确答案；玩家选完后直接进入下一题
- 全部题目结束后，只显示本次总结果和对应 GPA 修正

## 金钱与生活费

生活费每学期发放 `2` 次：

- 第 `1` 周
- 第 `4` 周

每周自动花费固定结算。

规则收束：

- 金钱是强约束资源
- 余额不足时，所有带花费行为直接禁用
- 不做债务系统
- 任意结算后金钱小于 `0`，立即触发中途失败结局 `生活费断裂`
- 金钱等于 `0` 不触发 `生活费断裂`，但所有带花费行为仍会因余额不足被禁用
- 避免出现不可控的“强制扣费直接判死”文本事件

### 每周自动恢复

- 每周所有行动、事件、商店、实习和自动花费结算完成后，统一执行基础周恢复：`精力 +10`、`压力 -5`
- 周恢复固定存在，不因 `模型周 / 赶图周 / 考试周 / 毕业设计阶段` 改变
- 周恢复后仍处于高危状态时，只保留高危行动限制，不额外追加兜底恢复

## 商店系统

商店为常驻轻量系统，不占行动次数。

定位：

- 补状态
- 小修正
- 轻量抗风险

边界：

- 不做库存
- 不做耐久
- 不做背包
- 不做 build 系统
- 强功能商品按学期限购
- 模型材料不限购买次数，金钱是主要限制
- 模型材料只能在模型周及其之前为当前课题购买；每次购买都立刻扣钱并结算
- 模型材料重复购买时，数值和风险权重修正正常累加，具体截断规则以 `numbers.md` 为准
- 商店商品只改数值，不改已经确定的商品内容和商品数量
- 消耗型补给按周限购，成长型商品按学期限购，强功能商品按学年或学期限购
- 商店只能做轻量成长和风险修正，不能替代行动系统成为主养成渠道
- 永久型、一学年型与下周生效型商品效果，都在周结算时自动读取
- 商店长期周恢复类效果合计每周最多生效：精力 `+3`，压力 `-5`；超过上限的部分在周结算时截断
- `星巴克会员周卡` 是唯一允许临时增加下周行动次数的商店商品
- `疯狂星期四` 每周限购 `1` 次。
- `红牛` 每周限购 `1` 次，定位为高精力恢复、高压力增加的冲刺补给。
- `眼药水` 每学期限购 `1` 次，购买后本学期每周固定小幅恢复精力并降低压力。
- `健身房年卡` 价格固定为 `3000`，每学年限购 `1` 次。
- `健身房年卡` 本学年内把精力临时上限提高到 `105`，并只修正 `健身运动` 的精力消耗；不增加行动次数，也不把商店变成主养成渠道。
- 不设置推荐商店；商店只展示完整商品清单、分类和可购买状态。

## 事件系统

当前事件文档按“随机事件体系”组织。

分池固定为：

- 普通随机事件
- 交互事件
- 模型周事件
- 万里路事件

课堂事件与导师偏移仍作为语境标签，不单独开池。
不再设置 `周结算事件`；一周结束时若需要保底触发，只从普通随机事件或交互事件中抽取。

### 触发原则

随机事件采用：

1. 先判定本周触发次数
2. 再判定事件池
3. 再按权重抽取

事件数量、学年目标、分池比例、连续触发目标和阶段权重统一维护在 `numbers.md` 的“事件权重与次数目标”。`systems.md` 只保留流程口径：

- 普通正式周有事件上限；万里路事件由玩家主动前往地点后必定触发，不计入普通周事件上限。
- 前期更容易触发日常与交互事件，后期逐步降低打断感。
- 固定流程事件和暑假写生节点不计入单局随机事件目标。
- 若玩家很少使用万里路，万里路目标差额由普通随机事件或交互事件自然补足。

### 结构原则

- 每条事件必须可落在既有系统内
- 不新增隐藏主数值
- 事件默认不直接改 `GPA / 作品集 / 个人简历 / 竞赛结果`
- 所有带选项事件统一归入交互事件
- 高贴脸重复事件控制在很小白名单内

### 结构化字段

事件配置字段口径统一维护在 `numbers.md` 的“程序落地格式”。流程层只要求：

字段规则：

- 标题重复不视为错误，但 `event_id` 必须唯一
- `event_id` 发布后保持稳定，标题和文案可以调整，程序主键不随文案变化
- `tags` 不新增玩法，只服务权重、冷却和内容筛选
- `context_tags` 是语境，不是新事件池
- `trigger_rules` 只描述触发条件与覆盖规则，不承载文本内容与数值结果
- 普通随机事件不写 `options`
- 交互事件不写隐藏概率树，只写显式选项

### 重复与冷却

- 同标题 `4` 周内不重复
- 首版只读取结构化字段 `base_weight / cooldown_weeks / repeatable / phase / tags`；语义重复与同类过密暂不另设公式。

## 建筑生的万里路

- 大二上开始开放；满足行动次数和金钱条件时可用

- 每学年最多触发 `4` 次
- 玩家按固定顺序推进地点，不随机跳点
- 玩家选择前往当前地点后，必定触发对应万里路事件
- 万里路事件的单次数值收益读取 `numbers.md` 的“万里路事件结算表”，用来匹配其 `2` 次行动和 `2500` 元代价
- 万里路事件不计入“普通正式周最多 `2` 次事件”的上限
- 第一次在模型周或赶图周使用万里路时，触发成长成就 `忙里偷闲`
- 完成全部 `12` 个地点后，万里路入口关闭
- 万里路只进入简历与成长路径，不进入作品集

- 每个写生事件只提供 `好好画 / 好好玩` 两个显式选项，选项结果统一走结算器；具体文本见 `events.md`，数值见 `numbers.md`。

## 作品集、竞赛、简历

### 作品集

- 作品集只保留一个总分，不拆子分。
- 作品集总分不设固定上限，首版按约 `0-900` 的体感区间校准。
- 作品集总分由已完成且最终图纸评级为 `D` 及以上课题的作品分累加形成。
- `F` 评级作品不进入作品集。
- 大一至大四每次正式评图结算后入库；大五毕业设计只在大五下最终答辩后入库 `1` 次，大五上期中汇报不重复计入。
- 竞赛、实习、万里路、商店和课程不直接增加作品集总分。

### 竞赛

- 竞赛素材只来自课程设计作业
- 只有满足对应赛事门槛的作品可以投稿
- 竞赛每学期最多投稿 `1` 次
- 同一作品只能投稿 `1` 次；这里的“作品”默认指一份已完成的课程设计作业
- 同一课题不能重复投向多个竞赛，本质上等同于“单作品唯一投稿”
- 奖项只保留：
  - 未获奖
  - 三等奖
  - 二等奖
  - 一等奖
- 竞赛不直接增加作品集
- 图纸等级只作为竞赛投稿门槛；获奖概率直接读取作品分
- `S` 级作品投稿对应赛事时必定获得一等奖；其他作品分区间均保留未获奖概率
- 竞赛可产生奖金，但奖金不反向增加作品集分数
- 竞赛结果在投稿后立即公开，不受学期结算时点限制
- 玩家可以随时查看自己的竞赛投稿记录与获奖情况
- 竞赛结果写入个人简历时只记录“赛事名 + 奖项 + 当次获奖经历”，不做简历总分
- 竞赛固定读取四项指定赛事：`霍普杯国际大学生建筑设计竞赛 / 台达杯国际太阳能建筑设计竞赛 / 台湾国际学生创意设计大赛 / 东南·中国建筑新人赛`
- `东南·中国建筑新人赛` 只允许大一、大二、大三阶段的课程设计作业投稿。
- 竞赛结果公开时必须弹出结果 UI，只显示奖项和对应结果文案；获奖时另显示统一奖金提示：`竞赛奖金 ￥{amount}`。
- 竞赛进入个人简历和成长成就的记录仍在后台结算，但不再单独弹出“简历写入提示”或“竞赛成就提示”。

### 个人简历

- 简历只做标签记录面板
- 不做简历总分系统
- 竞赛进入简历时，只记录是否投稿、赛事名与获奖档位
- 竞赛系统本身需要提供独立的“投稿记录 / 获奖情况”查看入口，不把查看需求全压到简历页
- 后期路线优先读取“有没有某类经历”

## 实习系统

- 实习入口从大一开始开放
- 实习入口未满足最低门槛时仍在 UI 中展示，并提示触发条件
- 实习只读取设计水平与软件技术门槛
- 实习工作池读取建筑相关工作池，不包括转行岗位池
- 每学期最多申请 `1` 次
- 实习申请不消耗行动次数
- 实习申请采用概率判定；录取概率表统一维护在 `numbers.md`
- 同学期申请失败后不可重投
- 每学期最多只保留 `1` 段实习
- 任意时刻只能有 `1` 段进行中的实习；已有进行中实习时，其他实习申请入口置灰。
- 实习录取后持续 `3` 周
- 实习允许跨学期继承剩余周数
- 已完成实习的单位不能再次申请
- 实习持续期间不占行动槽，但每周固定结算收益与代价
- 实习期间每周读取 `1` 条对应档位的短事件；短事件承担过程反馈，并保留对应轻量属性加成
- 实习结束时不再额外弹出“入职”“结束总结”或“简历写入”类提示；单位、持续周数、工资总额、属性变化和简历写入只作为结构化记录留在实习记录与个人简历中
- 实习可与正常学期并存；万里路不再占用整周，因此可与实习持续效果并存
- 实习申请成功只弹出提示：`你收到了一封实习录取邮件。`
- 实习申请失败只弹出提示：`你盯着那封拒绝信看了很久。`
- 实习完成后按档位累计 `实习价值`：普通实习 `+1`，强事务所实习 `+2`，名企实习 `+3`
- 建筑工作路线读取累计 `实习价值`；实习申请进阶读取最高已完成实习档位，不能用累计值替代最高档位

实习的作用固定为：

- 金钱收益
- 表格内的固定属性成长
- 简历标签
- 累计 `实习价值`
- 后期路线门槛读取

## 后期路线

从大四上第 `1` 周开始，系统提醒玩家可以准备后期方向，但不强制选择。大四阶段玩家可以先设置“路线意向”，路线意向可以修改；只有大五阶段玩家完成路线报名、申请、投递、考试或确认后，才形成“路线正式参与”记录。

规则：

- 未正式参与方向时，玩家仍可继续周行动、实习申请、竞赛投稿、商店和常规推进。
- 未正式参与方向时，路线报名、路线考试、路线投递和路线申请不会自动发生。
- 若玩家直到毕业仍未正式参与任何后期方向，结局系统跳过路线结果，读取普通状态毕业结局。
- 路线意向可以改；路线正式参与后不能改选，也不能跨方向补报名。
- 每局最多正式参与 `1` 个后期方向。
- 可正式参与方向为：`保研 / 考研 / 留学 / 选调 / 考公 / 考编 / 建筑工作 / 转行`。
- 选择 `转行` 时必须同步选择具体转行岗位，结局触发必须读取这个具体岗位。
- 选择 `选调 / 考公 / 考编` 时只能读取对应体制内子方向，不能在结算时互相串线。
- 选择 `考研 / 选调 / 考公 / 考编` 后，后续若数值或考试没有达到要求，只读取该方向的失败或回落结局，不再跳到工作、转行或其他路线。
- 选择 `建筑工作` 后，必须同步确认 `1` 个投递岗位；只读取该岗位成功结局或求职待定，不读取其他建筑岗位、转行岗位或升学结局；是否求职成功不提前公开，只在最终结局结算时揭示。
- 选择 `保研 / 留学` 后，只读取对应申请结果；失败后不自动切换到考研或工作成功结局。
- 正式参与 `保研` 时必须同步确认申请档位：`梦中情校`、`建筑老八校 / 其他 985`、`本校 / 211` 三档之一。
- 正式参与 `考研` 时必须同步确认报考档位：`梦中情校 / 建筑老八校 / 其他 985、211 / 普通一本院校` 中的对应档位。
- 正式参与 `留学` 时必须同步确认申请院校。
- 选择 `转行` 后，只读取对应转行岗位结果；是否转行成功不提前公开，只在最终结局结算时揭示。
- `转行` 下的 `创业` 是特殊高门槛入口：只有六项角色属性全部 `>= 90` 时才可选择；未满足时入口置灰并显示缺失属性。
- 玩家选择 `创业` 且满足门槛后，先弹出交互弹窗 `恶魔的契约`。玩家选择签下契约后才记录创业方向成功；选择放弃则保留为转行失败结果。
- 路线考试、申请、投递和内部判定结果只写入内部状态与日志，不提前展示给玩家。
- 玩家完成当次路线考试（保研复试、考研初试、考公、考编、选调）或提交留学申请后，统一弹出等待结果提示；文案读取 `content-plan.md` 的“路线等待结果弹窗文案”。
- 最终路线结果必须由统一结局解析器读取，其他系统不能提前写死人生结局。

正式参与任意路线后，其他路线入口关闭；UI 可保留入口但必须置灰，并提示已选择的正式路线。建筑工作与转行的简历投递只能投 `1` 个地方。

大五阶段首次进行 `建筑工作` 或 `转行` 的非创业岗位简历投递前，UI 弹出确认提示；文案读取 `content-plan.md` 的“路线投递确认弹窗文案”。`创业` 不属于简历投递，不弹出该确认提示。

路线组固定为：

- 升学路线组
- 体制内路线组
- 建筑工作路线组
- 转行路线组

### 路线收束原则

- 保研与考研归入升学路线组
- 考公、考编、选调归入体制内路线组
- 建筑工作和转行在大五阶段通过投递或确认形成正式参与记录，再按该方向做结果判定
- 留学只做申请判定，不展开复试玩法
- 建筑工作失败后保留 `求职待定——回去等通知吧` 兜底路径
- 所有成功路线最终只产出 `1` 个主结局
- 状态毕业结局是完整毕业但路线未成功玩家的最终保底

### 路线考试与答题

- 保研复试与考研初试共用 `升学专业题库`。
- 升学专业题库固定 `40` 道单选题，每次抽取 `10` 道：`6` 道简单题、`4` 道困难题。
- 考公、考编、选调共用 `行测题库`。
- 行测题库固定 `30` 道单选题，每次抽取 `10` 道：`6` 道简单题、`4` 道困难题。
- 路线考试答题过程中不显示单题对错，也不显示正确答案；玩家选完后直接进入下一题。
- 大五上评图结束后，若玩家已正式参与 `保研 / 考研 / 选调 / 考公 / 考编` 且达到对应门槛，则强制进入对应考试阶段。
- 路线考试结束后只记录内部结果，不向玩家显示总分、答对题数、是否达线，也不提前公布成功或失败。
- 路线考试完成后，必须显示统一等待结果弹窗；文案读取 `content-plan.md` 的“路线等待结果弹窗文案”。
- 路线考试结果统一保留到大五下毕业设计答辩结束后的结局结算阶段再揭示。
- 升学与行测考试都采用“`10` 题隐藏答题结果 -> 内部折算成绩”的方式；具体折算分与分数线统一维护在 `numbers.md`。
- 大五上评图结束后，如果已正式参与 `保研 / 考研 / 选调 / 考公 / 考编` 但未达到对应门槛，则不进入考试，只缓存为该路线未达报名或申请条件。

### 毕业设计门槛

- 大五上和大五下均为毕业设计阶段，并共同承载同一个毕业设计。
- 大五毕业设计的进度与作品质量上限均为 `200`。
- 大五上毕业设计期中汇报前，毕业设计进度必须达到 `100`；大五上结束后进度与作品质量不清零，继续继承到大五下。
- 大五上毕业设计期中汇报的作品质量按梯度判定，至少必须达到不被评为 `F` 的门槛。
- 顺利毕业必须完成毕业设计。
- 毕业设计完成条件为：大五下毕业设计评图 `D` 及以上，且毕业设计最终进度和作品质量达到数值规则要求。
- 大五下毕业设计答辩结束后，直接触发结局系统结算，不再额外延后。
- 若完整读到毕业但未完成毕业设计，直接读取 `毕业失败——延毕`，不读取其他成功路线结局。

### 状态收尾

- 成功通关只追加 `1` 句状态收尾。
- 状态收尾只保留 `普通 / 沉重` 两类。
- 普通：通关时不满足沉重。
- 沉重：通关时 `精力 < 35` 或 `压力 > 75`。

### 强路线可达性

- 强路线指明显优于普通毕业的高门槛成功结局；具体分档与调参目标以 `numbers.md` 的“强路线稀有度”为准。
- 强路线不要求多数玩家自然达成
- 普通通关必须稳定可达
- 强路线失败不导致中途失败，只读取该方向专属回落；若该方向没有专属回落且完成毕业设计，则读取状态毕业结局

## 失败与通关

### 中途失败

中途失败只保留四条：

- 生活费断裂
- 压力失控
- 连续挂科退学
- 被迫停学

### 成功通关

- 只有完整读完五年、未中途失败且完成毕业设计，才算成功通关
- 成功通关只读取 `1` 个主结局 + `1` 句状态收尾

### 结局读取优先级

最终结局只在统一结局解析器中读取，优先级固定为：

1. 中途失败
2. 毕业失败——延毕
3. 已正式参与路线结果
4. 状态毕业

强路线只是高门槛成功结果，不是额外失败条件。
