# 游戏系统架构

## 总体结构

游戏按 `周 -> 学期 -> 学年 -> 五年` 推进。

- 一学期：`6` 周
- 一学年：`12` 周
- 完整流程：`5` 年，共 `60` 周
- 每周固定：`3` 次行动
- 第 `5` 周：模型周
- 第 `6` 周：赶图周
- 赶图周结束后：进入评图阶段

## 角色系统

- 开局随机 `3` 张角色卡
- 玩家可免费重抽 `1` 次
- 最终选择 `1` 张角色卡开始游戏
- 角色额外记录学历背景，用于塑造开局差异与部分路线语境
- 每个角色固定拥有：
  - 初始精力
  - 初始压力
  - 初始金钱
  - 六项角色属性
  - `1` 个被动特质
  - `1` 个专属技能

规则收束：

- 不新增额外行动槽
- 不新增角色专属资源
- 被动特质尽量只修改既有数值规则
- 专属技能统一占用 `1` 次行动机会，统一冷却 `10` 周

## 导师系统

- 每学年重新分配导师
- 每次从 `3` 位导师中选择 `1` 位
- 不设置导师认可度
- 每位导师每学期给 `1` 个阶段任务
- 阶段任务只统计当学期行为
- 阶段任务统一在评图前结算
- 阶段任务只做“完成 / 未完成”判定，不做半完成档
- 阶段任务文本在学期第 `1` 周完整公开

导师影响固定为：

- 阶段任务
- 汇报偏好
- 事件倾向
- 路线倾向

### 阶段任务开放原则

- 早期学期不出现依赖高阶系统的任务
- 依赖 `竞赛投稿 / 名企实习 / 本学期 A` 的任务，只在对应系统已经开放后出现
- 不能要求玩家在“评图后”才能完成当学期任务

## 课题系统

每学期课题只保留两个临时值：

- 进度：`0-100`
- 作品质量：`0-100`

学期结束后清零。

进度主要来自：

- 正常速度画图
- 爆肝通宵赶图
- 部分随机事件
- 部分交互事件
- 部分专属技能

作品质量主要来自：

- 方案推敲
- 场地调研
- 模型周材料选择
- 模型周事件
- 部分随机事件
- 部分交互事件
- 部分专属技能

## 周行动系统

### 固定行动数

- 每周固定 `3` 次行动
- 不因角色、导师、被动、技能、副系统增加或减少

### 不占行动次数的系统

- 选课
- 商店购物
- 竞赛投稿判定
- 实习申请判定
- 学期末课程题
- 年度雅思考试

### 汇报策略

- 汇报策略只在评图阶段选择
- 不提前展开为独立主玩法

## 评图系统

评图阶段不再触发普通随机事件。

评图只读取四项：

- 进度
- 作品质量
- 汇报策略结果
- 导师任务修正

评图等级固定为：

- A
- B
- C
- D
- F

规则：

- `D` 视为过关
- `F` 视为挂科
- 连续 `2` 次 `F` 直接触发 `连续挂科退学`

## GPA 系统

- GPA 范围：`0.0-4.0`
- 开局显示 GPA：`4.0`
- GPA 只由学期评图结果与大学课程修正构成
- GPA 服务于后期路线，不反向改变每周行动结构

## 大学课程系统

- 每学期开始可从课程目录中选择 `1` 门课
- 课程选择不占每周行动次数
- 大学课程不做独立玩法
- 大学课程的作用固定为：
  - 属性倾向
  - 事件倾向
  - GPA 学期修正

学期结束：

- 回答 `3` 道对应课程题
- 直接结算课程修正
- 答错时直接显示正确答案

## 金钱与生活费

生活费每学期发放 `2` 次：

- 第 `1` 周
- 第 `4` 周

每周自动花费固定结算。

规则收束：

- 金钱是强约束资源
- 余额不足时，所有带花费行为直接禁用
- 不做债务系统
- 任意结算后金钱归零，立即触发中途失败结局 `生活费断裂`
- 避免出现不可控的“强制扣费直接判死”文本事件

## 商店系统

商店为常驻轻量系统，不占行动次数。

定位：

- 补状态
- 小修正
- 轻量抗风险

边界：

- 不做库存
- 不做耐久
- 不做背包
- 不做 build 系统
- 强功能商品按学期限购
- 模型材料每学期只允许购买 `1` 次

## 事件系统

当前事件文档按“随机事件体系”组织。

分池固定为：

- 普通随机事件
- 交互事件
- 模型周事件
- 万里路事件
- 周结算事件
- 路线事件
- 毕业事件

课堂事件与导师偏移仍作为语境标签，不单独开池。

### 触发原则

随机事件采用：

1. 先判定本周触发次数
2. 再判定事件池
3. 再按权重抽取

### 结构原则

- 每条事件必须可落在既有系统内
- 不新增隐藏主数值
- 事件默认不直接改 `GPA / 作品集 / 个人简历 / 竞赛结果`
- 所有带选项事件统一归入交互事件
- 高贴脸重复事件控制在很小白名单内

### 结构化字段

开发前每条事件必须补齐统一字段，字段口径固定如下：

- `event_id`：唯一主键，程序只读取它，不读取标题作为主键
- `title`：展示标题，允许重复
- `pool`：事件池，只允许 `normal / interactive / model / travel`
- `phase`：可触发阶段，如 `normal_week / model_week / travel / late_route`
- `sentiment`：体验方向，只允许 `positive / negative / neutral / mixed`
- `tags`：内容标签，最多 `3` 个
- `context_tags`：语境标签，如导师、课程、AI、万里路地点
- `base_weight`：基础权重，默认 `100`
- `cooldown_weeks`：冷却周数，默认 `4`
- `repeatable`：是否允许重复，默认 `false`
- `ai_related`：是否属于 AI 相关事件，默认 `false`
- `ai_experience_delta`：是否增加 AI 相关经历，默认 `0`
- `result`：无选项事件的数值结算
- `options`：交互事件的选项与各自数值结算

字段规则：

- 标题重复不视为错误，但 `event_id` 必须唯一
- `tags` 不新增玩法，只服务权重、冷却和内容筛选
- `context_tags` 是语境，不是新事件池
- 普通随机事件不写 `options`
- 交互事件不写隐藏概率树，只写显式选项

### 重复与冷却

- 同标题 `4` 周内不重复
- 同语义高频事件按冷却递减
- 同类事件过密时自动降权

## 建筑生的万里路

- 大二上开启
- 每次消耗 `1` 整周
- 消耗该周全部 `3` 次行动
- 不可落在模型周或赶图周
- 每学年最多触发 `3` 次
- 玩家按固定顺序推进地点，不随机跳点
- 万里路只进入简历与成长路径，不进入作品集

## 暑假写生系统

- 只保留 `2` 个固定节点
- 不占普通周行动次数
- 只提供属性与事件收益
- 不设置单独失败条件

## AI 相关语境

- 从大二上开始开放
- AI 不再拆成独立主系统
- AI 只通过以下内容体现：
  - AI 相关事件
  - AI 态度
  - AI 相关经历
  - AI 新职业路线

规则收束：

- AI 态度不做复杂成长面板
- AI 态度只通过事件逐步形成
- 玩家即使完全不碰 AI，也必须能完整通关
- AI 相关经历是经历计数，不是第二套能力分
- AI 相关经历只由关键事件、关键选择和少数里程碑产生
- 同一事件重复触发时，不重复增加 AI 相关经历

## 作品集、竞赛、简历

### 作品集

- 作品集只保留一个总值 `0-100`
- 不拆子分
- 主要由课题评图结果增长

### 竞赛

- 竞赛素材只来自课程设计作业
- 只有 `B / A` 课题可以投稿
- 每学期最多 `1` 次竞赛投稿判定
- 奖项只保留：
  - 未获奖
  - 三等奖
  - 二等奖
  - 一等奖
- 竞赛不直接增加作品集

### 个人简历

- 简历只做标签记录面板
- 不做简历总分系统
- 后期路线优先读取“有没有某类经历”

## 实习系统

- 实习入口从大二上开放
- 实习只读取设计水平与软件技术门槛
- 每学期最多申请 `1` 次
- 同学期申请失败后不可重投
- 每学期最多只保留 `1` 段实习
- 实习录取后固定持续 `4` 周
- 实习持续期间不占行动槽，但每周固定结算收益与代价
- 实习可与正常学期并存，但不能与万里路同周叠加

实习的作用固定为：

- 金钱收益
- 少量属性成长
- 简历标签
- 后期路线加权

## 后期路线

从大五阶段统一进入后期路线判定。

路线组固定为：

- 升学路线组
- 体制内路线组
- 建筑工作路线组
- 转行路线组

### 路线收束原则

- 保研与考研归入升学路线组
- 考公、考编、选调归入体制内路线组
- 秋招、工作、转行集中在大五阶段判定
- 留学只做申请判定，不展开额外子玩法
- 秋招失败后保留兜底路径
- 所有成功路线最终只产出 `1` 个主结局
- `普通但稳定` 是所有未出局玩家的成功通关保底

### 强路线可达性

- 强路线指 `保研直升 / 海外留学 / 选调回乡 / 顶级事务所 / AI 新职业` 这类明显优于普通毕业的结局
- 强路线不要求多数玩家自然达成
- 普通通关必须稳定可达
- 强路线失败不导致失败结局，只回落到下一优先级路线或 `普通但稳定`

## 失败与通关

### 中途失败

中途失败只保留四条：

- 生活费断裂
- 压力失控
- 连续挂科退学
- 被迫停学

### 成功通关

- 只要完整读完五年且未中途失败，就算成功通关
- 成功通关只读取 `1` 个主结局 + `1` 句状态收尾

## 文档边界

- `systems.md` 只写系统规则与流程
- 具体数值写入 `numbers.md`
- 具体内容池写入 `content-plan.md`
- 具体事件写入 `events.md`
- 具体结局写入 `endings.md`
