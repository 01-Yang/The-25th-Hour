# 游戏系统架构

## 总体结构

游戏按 `周 -> 学期 -> 学年 -> 五年` 推进。

- 一学期：`6` 周
- 一学年：`12` 周
- 完整流程：`5` 年，共 `60` 周
- 每周固定：`3` 次行动
- 第 `5` 周：模型周
- 第 `6` 周：赶图周
- 赶图周结束后：进入评图阶段

首版体验定位：

- 游戏是“建筑生人生模拟器”，不是纯建筑设计训练模拟器。
- 建筑专业负责承载时间分配、压力、成长和结局，不追求完整复刻真实教学流程。
- 每周核心压力来自固定 `3` 次行动机会的取舍，不来自复杂菜单、隐藏系统或多路线并行管理。
- 普通通关应稳定可达，强路线保持稀有，中途失败要有明确预警。

### 程序状态机

开发实现必须以显式阶段状态机推进，不允许由 UI 页面各自决定流程。

推荐阶段顺序：

1. 开局角色选择
2. 固定流程事件
3. 学期开始
4. 周开始
5. 周行动
6. 周结算
7. 评图阶段
8. 学期结算
9. 年度切换
10. 后期路线锁定
11. 结局结算

规则：

- 游戏状态必须写入统一 `GameState`，UI 只读状态并发起操作。
- 行动、事件、商店、实习、评图、路线和结局都只能通过统一结算器修改状态。
- 每次结算都必须产出结构化数值变化和结算日志。
- 每局生成固定随机种子；随机事件、考试抽题、概率录取和投递结果都必须可复现。
- 自动存档点固定为：每周开始、每周结束、评图前、评图后、路线锁定后、结局结算前。

## 角色系统

- 开局随机 `3` 张角色卡
- 玩家可免费重抽 `1` 次
- 最终选择 `1` 张角色卡开始游戏
- 角色额外记录学历背景，用于塑造开局差异与部分路线语境
- 每个角色固定拥有：
  - 初始精力
  - 初始压力
  - 初始金钱
  - 六项角色属性
  - `1` 个被动特质
  - `1` 个专属技能

规则收束：

- 所有角色初始精力固定为 `100`
- 不新增额外行动槽
- 不新增角色专属资源
- 被动特质尽量只修改既有数值规则
- 专属技能统一占用 `1` 次行动机会，统一冷却 `10` 周

## 导师系统

- 军训结束后进行第一次导师选择
- 之后每个新学年重新选择导师
- 每次从 `3` 位导师中选择 `1` 位
- 不设置导师认可度
- 每位导师每学期给 `1` 个阶段任务
- 阶段任务只统计当学期行为
- 阶段任务统一在评图前结算
- 阶段任务只做“完成 / 未完成”判定，不做半完成档
- 阶段任务文本在学期第 `1` 周完整公开

导师影响固定为：

- 阶段任务
- 汇报偏好
- 事件倾向
- 路线倾向

### 阶段任务开放原则

- 早期学期不出现依赖高阶系统的任务
- 依赖 `竞赛投稿 / 名企实习` 等高阶系统的任务，只在对应系统已经开放后出现
- 不能要求玩家在“评图后”才能完成当学期任务
- 不能依赖当学期最终评图等级；如需表现“稳定通过”，只读取评图前可判定的过程指标或已完成学期记录

## 课题系统

每学期课题只保留两个临时值：

- 进度：`0-100`
- 作品质量：`0-100`

学期结束后清零。

例外：

- 大五上和大五下视为同一个 **毕业设计**，进度与作品质量不在大五上结束后清零。
- 大五毕业设计的进度与作品质量上限均为 `200`。
- 大五上为毕业设计期中汇报，期中汇报前毕业设计进度必须达到 `100`，作品质量至少达到不被评为 `F` 的门槛。

进度主要来自：

- 正常速度画图
- 爆肝通宵赶图
- 部分随机事件
- 部分交互事件
- 部分专属技能

作品质量主要来自：

- 方案推敲
- 场地调研
- 模型周材料选择
- 模型周事件
- 部分随机事件
- 部分交互事件
- 部分专属技能

联动原则：

- `正常速度画图` 与 `爆肝通宵赶图` 主负责 `进度`；所有带 `基础进度` 的常驻行动都读取 `软件技术` 档位修正。
- 普通学期中，约 `6-7` 次正常速度画图应能补满进度；约 `3-4` 次爆肝通宵赶图应能补满进度，但状态代价显著更高。
- 事件、导师任务和角色技能允许造成进度加减，因此实际画图次数可以围绕目标压力浮动。
- `方案推敲` 与 `场地调研` 主负责 `作品质量`，并固定附带 `进度 +2`。
- 所有带 `基础作品质量` 的常驻行动都读取 `设计水平 + 创意审美` 的简单档位修正。
- `方案推敲` 更偏室内高压深化，消耗精力并增加压力；`场地调研` 每周最多 `1` 次，更耗精力和金钱，但略微降低压力。
- 进度与质量合计不应吃掉整个学期；普通过关路线应能留下行动给学习、运动、放松、社交、兼职和突发补救。
- 画图速度与质量成长拆开处理，避免“快”与“好”在同一行动里同时滚满。

## 周行动系统

### 固定行动数

- 每周固定 `3` 次行动
- 不因角色、导师、被动、技能、副系统增加或减少
- 唯一例外为商店道具 `星巴克会员周卡`：购买后仅使`下一周`额外获得 `1` 次行动，总计 `4` 次，且同一周内此效果不叠加

### 不占行动次数的系统

- 选课
- 商店购物
- 竞赛投稿判定
- 实习申请判定
- 学期末课程题
- 年度雅思考试

### 开局固定流程

- 选择角色并开始游戏后，先进入 `开学典礼` 固定事件。
- 开学典礼结束后进入 `军训` 固定事件。
- 开学典礼与军训期间，所有周行动、商店、竞赛、实习、路线报名入口均不可使用。
- 军训结束后显示 `建院生活开始提示`。
- 建院生活开始提示结束后，玩家从 `3` 位导师中选择 `1` 位作为大一导师。
- 导师选择完成后，正式进入大一上第 `1` 周。

### 行动次数限制

- `接设计外包` 达到能力门槛后即可选择，一周内不限次数，但每次都正常占用 `1` 次周行动并结算代价。
- `校外兼职` 需要低社交门槛，一周内不限次数，但每次都正常占用 `1` 次周行动并结算代价。
- `场地调研` 每周最多选择 `1` 次。
- 高危状态仍可禁用 `接设计外包 / 校外兼职`，高危限制优先级高于“一周内不限次数”。

### 行动可用性

所有行动入口统一由可用性解析器判断。

解析结果只允许三类：

- `可用`：玩家可以直接选择。
- `置灰`：入口可见，但因资源、阶段、高危状态、路线锁定或次数限制暂时不可用，并显示原因。
- `隐藏`：当前阶段不应展示该入口，例如未开放路线或已关闭的万里路入口。

UI 不直接复制条件逻辑，只展示解析器返回的状态和原因文案。

### 汇报策略

- 汇报策略只在评图阶段选择
- 不提前展开为独立主玩法
- 汇报策略不再修正 `进度` 或 `作品质量`
- 汇报策略在基础图纸评级产生后结算，只影响最终图纸评级、作业分和设计课绩点
- 汇报成功率由策略固定值决定，属性只作为可选门槛
- 基础图纸评级为 `F` 时仍展示汇报策略，文案口径为“尘埃还未落定”
- 只有 `求求你别挂我` 可以在基础图纸评级为 `F` 时低概率补救为 `D`，其余策略不能救硬门槛或低质量导致的 `F`

### 高危状态行动限制

当进入高危状态时，行动界面必须提示玩家当前风险，并限制部分行动选项。

- 精力高危：`精力 < 20`
- 压力高危：`压力 > 80`
- 精力高危时，禁用 `爆肝通宵赶图 / 接设计外包 / 校外兼职 / 万里路 / 实习开始` 等高精力消耗行动。
- 压力高危时，禁用 `爆肝通宵赶图 / 接设计外包 / 高压专属技能 / 高压交互选项` 等高压力行动。
- 高危状态下至少保留 `休养生息` 和可支付的低风险恢复手段，避免玩家无行动可选。
- 高危限制只限制行动选择，不直接触发结局；结局仍按失败阈值在结算节点读取。
- 若玩家选择的行动或事件选项在结算后导致精力小于 `0`，不额外弹出风险确认，直接触发 `被迫停学`。

## 评图系统

评图阶段不再触发普通随机事件。

评图采用“进度硬门槛 + 基础图纸评级 + 汇报策略修正最终评级”的三段制。

进度硬门槛：

- 大一至大四每学期课题进度上限为 `100`；评图前进度未达到 `100` 时，本次评图直接为 `F`。
- 大五上为毕业设计期中汇报；毕业设计进度未达到 `100` 时，本次评图直接为 `F`。
- 大五下为毕业设计答辩；毕业设计最终进度未达到毕业门槛时，视为未完成毕业设计。

质量硬门槛：

- 大一至大四作品质量未达到 `F` 门槛时，基础图纸评级为 `F`。
- 大五毕业设计作品质量按 `0-200` 记录，评图时折算为 `0-100` 的质量评图分；折算后未达到 `F` 门槛时，基础图纸评级为 `F`。
- 基础图纸评级先由作品质量评图分决定。
- 汇报策略不改变作品质量，只在基础图纸评级之后改变最终图纸评级、作业分和设计课绩点。
- 导师阶段任务和部分事件只允许在评图前修正作品质量、状态或 GPA，不再产生独立“评图总分”。

评图等级固定为：

- A
- B
- C
- D
- F

规则：

- `D` 视为过关
- `F` 视为挂科
- 每次评图结束后，系统必须立刻直接告诉玩家本次最终图纸评级是 `A / B / C / D / F` 中的哪一档，并显示本次作业分。
- 连续 `2` 次 `F` 直接触发 `连续挂科退学`

## GPA 系统

- GPA 范围：`0.0-4.0`
- 开局显示 GPA：`4.0`
- GPA 只由学期评图结果与大学课程修正构成
- GPA 服务于后期路线，不反向改变每周行动结构

## 大学课程系统

- 每学期开始可从课程目录中选择 `1` 门课
- 课程选择不占每周行动次数
- 大学课程不做独立玩法
- 大学课程的作用固定为：
  - 属性倾向
  - 事件倾向
  - GPA 学期修正

学期结束：

- 回答 `3` 道对应课程题
- 直接结算课程修正
- 答题过程中不显示单题对错，也不显示正确答案；玩家选完后直接进入下一题
- 全部题目结束后，只显示本次总结果和对应 GPA 修正

## 金钱与生活费

生活费每学期发放 `2` 次：

- 第 `1` 周
- 第 `4` 周

每周自动花费固定结算。

规则收束：

- 金钱是强约束资源
- 余额不足时，所有带花费行为直接禁用
- 不做债务系统
- 任意结算后金钱小于 `0`，立即触发中途失败结局 `生活费断裂`
- 金钱等于 `0` 不触发 `生活费断裂`，但所有带花费行为仍会因余额不足被禁用
- 避免出现不可控的“强制扣费直接判死”文本事件

### 每周自动恢复

- 每周所有行动、事件、商店、实习和自动花费结算完成后，统一执行 `精力 +12`、`压力 -10`
- 周恢复固定存在，不因 `模型周 / 赶图周 / 考试周 / 毕业设计阶段` 改变
- 周恢复后仍处于高危状态时，只保留高危行动限制，不额外追加兜底恢复

## 商店系统

商店为常驻轻量系统，不占行动次数。

定位：

- 补状态
- 小修正
- 轻量抗风险

边界：

- 不做库存
- 不做耐久
- 不做背包
- 不做 build 系统
- 强功能商品按学期限购
- 模型材料每学期只允许购买 `1` 次
- 商店商品只改数值，不改已经确定的商品内容和商品数量
- 消耗型补给按周限购，成长型商品按学期限购，强功能商品按学年或学期限购
- 商店只能做轻量成长和风险修正，不能替代行动系统成为主养成渠道
- 永久型、一学年型与下周生效型商品效果，都在周结算时自动读取
- `星巴克会员周卡` 是唯一允许临时增加下周行动次数的商店商品
- 不设置推荐商店；商店只展示完整商品清单、分类和可购买状态。

## 事件系统

当前事件文档按“随机事件体系”组织。

分池固定为：

- 普通随机事件
- 交互事件
- 模型周事件
- 万里路事件

课堂事件与导师偏移仍作为语境标签，不单独开池。
不再设置 `周结算事件`；一周结束时若需要保底触发，只从普通随机事件或交互事件中抽取。

### 触发原则

随机事件采用：

1. 先判定本周触发次数
2. 再判定事件池
3. 再按权重抽取

单局触发目标：

- 单局事件目标为 `90-95` 次，标准目标按 `92` 次校准。
- `92` 次事件发生在 `60` 个正式周里，包含普通随机事件、交互事件、模型周事件和万里路事件。
- 固定流程事件、暑假写生节点不计入这 `92` 次事件。

事件节奏规则：

- 普通正式周最多触发 `2` 次普通随机事件、交互事件或模型周事件；万里路事件由玩家主动前往地点后必定触发，不计入这个上限。
- 大一、大二启用周保底：若一周自然抽取后没有任何事件，则周末保底触发 `1` 次普通随机事件或交互事件。
- 大三后不做每周保底；连续 `2` 周没有任何事件时，下一周周末保底触发 `1` 次普通随机事件或交互事件。
- 普通随机事件与交互事件在体感上经常前后呼应，形成“日常事件 + 交互事件”的节奏。
- 连续 `2` 次都是普通随机事件的概率目标为 `15%`。
- 连续 `2` 次都是交互事件默认不作为目标节奏；若因保底或阶段限定出现，后续交互事件临时降权。
- 普通随机事件与交互事件的单局目标分别为 `46` 与 `34`；两者体感上经常前后呼应，但不按固定比例硬切。

学年目标次数：

| 学年 | 目标触发次数 | 周均体感 |
|---|---:|---:|
| 大一 | `22` | 约 `1.8` 次/周 |
| 大二 | `22` | 约 `1.8` 次/周 |
| 大三 | `18` | 约 `1.5` 次/周 |
| 大四 | `16` | 约 `1.3` 次/周 |
| 大五 | `14` | 约 `1.1` 次/周 |

事件池目标次数：

| 事件池 | 单局目标次数 |
|---|---:|
| 普通随机事件 | `46` |
| 交互事件 | `34` |
| 模型周事件 | `6` |
| 万里路事件 | `6` |

阶段权重：

- 大一、大二提高交互事件与普通随机事件的平均触发概率。
- 大一、大二的交互事件权重上调不只偏向社交、朋友、暧昧、娱乐类，也必须同步保留学业交互权重，避免早期体验变成纯生活事件。
- 大一、大二交互事件推荐权重方向为：学业交互约 `40%`，社交/朋友/暧昧约 `35%`，娱乐/恢复约 `25%`。
- 大三后逐步降低娱乐类权重，保留学业压力、能力成长、路线前置和状态管理事件。

### 结构原则

- 每条事件必须可落在既有系统内
- 不新增隐藏主数值
- 事件默认不直接改 `GPA / 作品集 / 个人简历 / 竞赛结果`
- 所有带选项事件统一归入交互事件
- 高贴脸重复事件控制在很小白名单内

### 结构化字段

开发前每条事件必须补齐统一字段，字段口径固定如下：

- `event_id`：唯一主键，程序只读取它，不读取标题作为主键
- `title`：展示标题，允许重复
- `pool`：事件池，只允许 `normal / interactive / model / travel`
- `phase`：可触发阶段，如 `normal_week / model_week / travel / late_route`
- `sentiment`：体验方向，只允许 `positive / negative / neutral / mixed`
- `tags`：内容标签，最多 `3` 个
- `context_tags`：语境标签，如导师、课程、AI、万里路地点
- `base_weight`：基础权重，默认 `100`
- `cooldown_weeks`：冷却周数，默认 `4`
- `repeatable`：是否允许重复，默认 `false`
- `ai_related`：是否属于 AI 相关事件，默认 `false`
- `ai_experience_delta`：是否增加 AI 相关经历，默认 `0`
- `trigger_rules`：触发限定与额外前置条件，如保底、周次限定、阶段限定、黑名单条件
- `result`：无选项事件的数值结算
- `options`：交互事件的选项与各自数值结算

字段规则：

- 标题重复不视为错误，但 `event_id` 必须唯一
- `event_id` 发布后保持稳定，标题和文案可以调整，程序主键不随文案变化
- `tags` 不新增玩法，只服务权重、冷却和内容筛选
- `context_tags` 是语境，不是新事件池
- `trigger_rules` 只描述触发条件与覆盖规则，不承载文本内容与数值结果
- 普通随机事件不写 `options`
- 交互事件不写隐藏概率树，只写显式选项

### 重复与冷却

- 同标题 `4` 周内不重复
- 同语义高频事件按冷却递减
- 同类事件过密时自动降权

## 建筑生的万里路

- 大二上开始即可在满足条件时使用
- 每次消耗 `2` 次行动机会
- 可在模型周或赶图周使用，仍正常消耗 `2` 次行动机会
- 每学年最多触发 `4` 次
- 玩家按固定顺序推进地点，不随机跳点
- 玩家选择前往当前地点后，必定触发对应万里路事件
- 万里路事件不计入“普通正式周最多 `2` 次事件”的上限
- 第一次在模型周或赶图周使用万里路时，触发成长成就 `忙里偷闲`
- 完成全部 `12` 个地点后，万里路入口关闭
- 万里路只进入简历与成长路径，不进入作品集

## 暑假写生系统

- 只保留 `2` 个固定节点
- 不占普通周行动次数
- 只提供属性与事件收益
- 不设置单独失败条件

## AI 相关语境

- 从大一下开始开放
- AI 不再拆成独立主系统
- AI 只通过以下内容体现：
  - AI 相关事件
  - AI 相关经历
  - AI 新职业路线

规则收束：

- 不保留 AI 立场设定，不做 AI 阵营或态度标签
- 玩家即使完全不碰 AI，也必须能完整通关
- AI 相关经历是经历计数，不是第二套能力分
- AI 相关经历只由关键事件、关键选择和少数里程碑产生
- 同一事件重复触发时，不重复增加 AI 相关经历

## 作品集、竞赛、简历

### 作品集

- 作品集只保留一个总值 `0-100`
- 不拆子分
- 主要由课题评图结果增长

### 竞赛

- 竞赛素材只来自课程设计作业
- 只有满足对应赛事门槛的作品可以投稿
- 竞赛每学期最多投稿 `1` 次
- 同一作品只能投稿 `1` 次；这里的“作品”默认指一份已完成的课程设计作业
- 同一课题不能重复投向多个竞赛，本质上等同于“单作品唯一投稿”
- 奖项只保留：
  - 未获奖
  - 三等奖
  - 二等奖
  - 一等奖
- 竞赛不直接增加作品集
- 是否得奖只看该作品的 `投稿总分`
- 竞赛可产生奖金，但奖金不反向增加作品集分数
- 竞赛结果在投稿后立即公开，不受学期结算时点限制
- 玩家可以随时查看自己的竞赛投稿记录与获奖情况
- 竞赛结果写入个人简历时只记录“赛事名 + 奖项 + 当次获奖经历”，不做简历总分
- 竞赛固定读取四项指定赛事：`霍普杯国际大学生建筑设计竞赛 / 台达杯国际太阳能建筑设计竞赛 / 台湾国际学生创意设计大赛 / 东南·中国建筑新人赛`

### 个人简历

- 简历只做标签记录面板
- 不做简历总分系统
- 竞赛进入简历时，只记录是否投稿、赛事名与获奖档位
- 竞赛系统本身需要提供独立的“投稿记录 / 获奖情况”查看入口，不把查看需求全压到简历页
- 后期路线优先读取“有没有某类经历”

## 实习系统

- 实习入口从大一开始开放
- 实习入口未满足最低门槛时仍在 UI 中展示，并提示触发条件
- 实习只读取设计水平与软件技术门槛
- 实习工作池读取建筑相关工作池，不包括转行岗位池
- 每学期最多申请 `1` 次
- 实习申请不消耗行动次数
- 实习申请采用概率判定；刚满足最低门槛时录取率较低，高度超过门槛时录取率达到 `100%`
- 同学期申请失败后不可重投
- 每学期最多只保留 `1` 段实习
- 实习录取后一般持续 `3` 周
- 实习允许跨学期继承剩余周数
- 已完成实习的单位不能再次申请
- 实习持续期间不占行动槽，但每周固定结算收益与代价
- 实习结束时必须显示实习总结，包括单位、持续周数、工资总额、属性变化和简历写入
- 实习可与正常学期并存；万里路不再占用整周，因此可与实习持续效果并存

实习的作用固定为：

- 金钱收益
- 少量属性成长
- 简历标签
- 后期路线加权

## 后期路线

从大四上第 `1` 周开始，玩家必须先锁定一个后期方向，然后才能继续进行本周行动。

规则：

- 未锁定方向时，不能进行任何周行动、实习申请、竞赛投稿、路线报名或商店外的推进操作。
- 每局只能锁定 `1` 个后期方向，锁定后不能改选，也不能跨方向补报名。
- 可锁定方向为：`保研 / 考研 / 留学 / 选调 / 考公 / 考编 / 建筑工作 / 转行`。
- 选择 `转行` 时必须同步选择具体转行岗位，结局触发必须读取这个具体岗位。
- 选择 `选调 / 考公 / 考编` 时只能读取对应体制内子方向，不能在结算时互相串线。
- 选择 `考研 / 选调 / 考公 / 考编` 后，后续若数值或考试没有达到要求，只读取该方向的失败或回落结局，不再跳到工作、转行或其他路线。
- 选择 `建筑工作` 后，只读取建筑就业相关结局或求职待定，不读取转行岗位结局；是否求职成功不提前公开，只在最终结局结算时揭示。
- 选择 `保研 / 留学` 后，只读取对应申请结果；失败后不自动切换到考研或工作成功结局。
- 选择 `保研` 时必须同步锁定申请档位：`梦中情校`、`建筑老八校 / 其他 985`、`本校 / 211` 三档之一。
- 选择 `考研` 时必须同步锁定报考档位：`梦中情校 / 建筑老八校 / 其他 985、211 / 普通一本院校` 中的对应档位。
- 选择 `留学` 时必须同步锁定申请院校。
- 选择 `转行` 后，只读取对应转行岗位结果；是否转行成功不提前公开，只在最终结局结算时揭示。
- 路线考试、申请、投递和概率结果只写入内部状态与日志，不提前展示给玩家。
- 最终路线结果必须由统一结局解析器读取，其他系统不能提前写死人生结局。

路线组固定为：

- 升学路线组
- 体制内路线组
- 建筑工作路线组
- 转行路线组

### 路线收束原则

- 保研与考研归入升学路线组
- 考公、考编、选调归入体制内路线组
- 秋招、工作、转行在大四上锁定方向，大五阶段只按已锁定方向做结果判定
- 留学只做申请判定，不展开复试玩法
- 秋招失败后保留兜底路径
- 所有成功路线最终只产出 `1` 个主结局
- 状态毕业结局是完整毕业但路线未成功玩家的最终保底

### 路线考试与答题

- 保研复试与考研初试共用 `升学专业题库`。
- 升学专业题库固定 `30` 道单选题，每次抽取 `10` 道：`6` 道简单题、`4` 道困难题。
- 考公、考编、选调共用 `行测题库`。
- 行测题库固定 `30` 道单选题，每次抽取 `10` 道：`6` 道简单题、`4` 道困难题。
- 路线考试答题过程中不显示单题对错，也不显示正确答案；玩家选完后直接进入下一题。
- 大五上评图结束后，若玩家已锁定 `保研 / 考研 / 选调 / 考公 / 考编` 且达到对应门槛，则强制进入对应考试阶段。
- 路线考试结束后只记录内部结果，不向玩家显示总分、答对题数、是否达线，也不提前公布成功或失败。
- 路线考试结果统一保留到大五下毕业设计答辩结束后的结局结算阶段再揭示。
- 升学与行测考试都采用“`10` 题隐藏答题结果 -> 内部折算成绩”的方式：玩家只做 `10` 题，系统内部再折算为更贴近现实的分数线。
- 升学专业考试折算分固定为：答对 `10 / 9 / 8 / 7 / 6` 题时，分别折算为 `410 / 395 / 380 / 365 / 350` 分；答对 `5` 题及以下统一视为 `350` 分以下。
- 行测考试折算分固定为：答对 `10 / 9 / 8 / 7 / 6` 题时，分别折算为 `150 / 145 / 140 / 135 / 130` 分；答对 `5` 题及以下统一视为 `130` 分以下。
- 大五上评图结束后，如果已锁定 `保研 / 考研 / 选调 / 考公 / 考编` 但未达到对应门槛，则不进入考试，只缓存为该路线未达报名或申请条件。

### 毕业设计门槛

- 大五上和大五下均为毕业设计阶段，并共同承载同一个毕业设计。
- 大五毕业设计的进度与作品质量上限均为 `200`。
- 大五上毕业设计期中汇报前，毕业设计进度必须达到 `100`；大五上结束后进度与作品质量不清零，继续继承到大五下。
- 大五上毕业设计期中汇报的作品质量按梯度判定，至少必须达到不被评为 `F` 的门槛。
- 顺利毕业必须完成毕业设计。
- 毕业设计完成条件为：大五下毕业设计评图 `D` 及以上，且毕业设计最终进度和作品质量达到数值规则要求。
- 大五下毕业设计答辩结束后，直接触发结局系统结算，不再额外延后。
- 若完整读到毕业但未完成毕业设计，直接读取 `没毕业` 结局，不读取其他成功路线结局。

### 强路线可达性

- 强路线指 `保研直升 / 海外留学 / 选调回乡 / 顶级事务所 / AI 新职业` 这类明显优于普通毕业的结局
- 强路线不要求多数玩家自然达成
- 普通通关必须稳定可达
- 强路线失败不导致中途失败，只读取该方向专属回落；若该方向没有专属回落且完成毕业设计，则读取状态毕业结局

## 失败与通关

### 中途失败

中途失败只保留四条：

- 生活费断裂
- 压力失控
- 连续挂科退学
- 被迫停学

### 成功通关

- 只有完整读完五年、未中途失败且完成毕业设计，才算成功通关
- 成功通关只读取 `1` 个主结局 + `1` 句状态收尾

### 结局读取优先级

最终结局只在统一结局解析器中读取，优先级固定为：

1. 中途失败
2. 没毕业
3. 已锁定路线结果
4. 状态毕业

强路线只是高门槛成功结果，不是额外失败条件。
