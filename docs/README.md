# 文档索引

当前文档目标是：少而清楚，方便继续讨论，也方便后续进入开发。

## 推荐阅读顺序

1. `../CONTEXT.md`：领域语言与术语统一口径。
2. `PRD.md`：产品目标、体验目标和系统边界。
3. `development-freeze.md`：已经确认不再回滚的主系统边界。
4. `systems.md`：游戏流程、系统结构和规则口径。
5. `numbers.md`：属性、行动、评图、竞赛、路线门槛等数值规则。

## 内容文档

- `world-and-tone.md`：世界观、情绪气质、叙事原则和 slogan。
- `content-plan.md`：角色、导师、课程、音乐、路线、岗位等内容池。
- `events.md`：事件原则、事件骨架和事件正文。
- `endings.md`：人生结局与成长成就。
- `question-banks.md`：大学课程、雅思、升学专业和行测题库。

## 开发与追溯

- `unresolved-issues.md`：只记录仍需实现验证、跑局模拟或后续调参的问题。
- `decision-log.md`：已合并的历史决策摘要与归档来源摘要。
- `adr/`：关键结构决策记录。

## 文档分工

- `systems.md` 写“规则怎么流转”。
- `numbers.md` 写“数值怎么结算”。
- `content-plan.md / events.md / endings.md / question-banks.md` 写“展示给玩家的内容和内容池”。
- `development-freeze.md` 写“哪些系统边界已经冻结”。
- `unresolved-issues.md` 写“还要验证什么”，不再保留已解决问题长列表。

## 优先级

若文档之间出现冲突，按以下顺序处理：

1. `development-freeze.md`
2. `systems.md`
3. `numbers.md`
4. 内容文档
5. `decision-log.md`
