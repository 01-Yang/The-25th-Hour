# 待验证问题

最后更新时间：`2026-05-22 +08:00`

当前没有阻塞开发前框架收敛的主系统问题。已确认的架构口径见 `development-freeze.md`，历史决策摘要见 `decision-log.md`。

## 实现验证

- 随机事件配置需要落成 `event_id / pool / phase / result 或 options / base_weight / cooldown_weeks` 等粗权重字段，并验证单局事件量稳定落在 `90-95` 次附近。
- 高危状态 UI 需要验证状态区呼吸、首次高危弹窗、行动置灰和原因提示，尤其是精力高危、压力高危、双高危三种情况。
- 大四上第 `1` 周路线提醒必须是非强制流程；未正式参与方向时，周行动、实习、竞赛投稿、商店和常规推进仍可继续，路线报名、考试、投递和申请不应自动发生。
- 大五路线考试阶段与大五下答辩后的结局结算之间，需要补齐“隐藏结果缓存 + 统一揭示”的实现流程。
- 保研复试、考研初试、行测考试都需要实现“选完直接下一题、单题不反馈对错”的答题交互。
- 留学申请、实习录取和岗位投递都需要走固定随机种子；玩家只看到模糊分档或等待结果，不显示精确概率和内部结果。
- `星巴克会员周卡` 是唯一的下周 `+1` 行动次数例外，需要验证同周多次购买不能叠加。
- `基层服务` 判定需要放在 `继续备考` 前，避免被失败兜底吞掉。

## 跑局调参

- 首轮跑局目标：普通通关率 `65%-80%`，中途失败率 `10%-20%`，强路线总触发率 `8%-15%`。
- 画图、赶图与质量类行动需要跑一轮学期模拟，确认 `软件技术` 和 `设计水平 + 创意审美` 的修正不会让中后期收益失衡。
- 六项属性轻衰减、商店成长商品和实习收益需要验证，避免大二、大三就把关键属性推满。
- 四项竞赛需要模拟验证三等奖、二等奖、一等奖的实际出现率是否符合体感。
- 导师阶段任务需要验证失败惩罚体感，重点观察戈老师 `GPA学期修正 -0.50`、林老师和韩老师压力 `+10`。
- `没毕业` 结局暂不扩写；跑完整局后再判断是否需要补充情绪重量。

## 首版闭环

- 首版实现先做纯文本或低 UI 最小可玩闭环，验证 `GameState -> 状态机 -> 结算器 -> 毕业典礼 -> 毕业放映 -> 结局解析器` 能完整从选角色跑到结局。
- 开发第一阶段需要在固定 seed 下稳定复现一局，并输出完整结算日志、最终结局和关键统计。
