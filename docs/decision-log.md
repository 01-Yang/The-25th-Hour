# 决策日志

本文只保留已经确认、但有必要追溯来源的历史决策摘要。当前规则优先级始终是：`development-freeze.md` -> `systems.md` -> `numbers.md` -> 内容文档。

## 2026-05-16 系统收敛

- 首版冻结为 `5` 年、`60` 周、每周 `3` 次行动、每学期 `1` 次评图的主循环。
- 后期路线改为互斥读取：升学、体制内、建筑工作、转行最终只读取 `1` 个主结局。
- 考试统一为隐藏答题结果，再由系统内部折算现实分数线。
- 商店只做轻量修正，不做库存、背包、耐久或 build 系统。
- 开发必须使用统一 `GameState`、显式状态机、统一结算器、固定随机种子和可复现结算日志。

## 2026-05-18 事件节奏

- 单局随机事件目标调整为 `90-95` 次，标准目标按 `92` 次校准。
- 固定流程事件和暑假写生节点不计入随机事件目标。
- 大一、大二事件密度更高；大四增加焦虑语境，大五增加感动语境。
- 事件池收束为普通随机事件、交互事件、模型周事件、万里路事件；AI、导师、课程只作为语境标签。
- AI 相关事件从大一下开始出现，但不新增独立 AI 主系统。

## 2026-05-21 数值与路线收敛

- 毕业尾段固定为：大五答辩 -> 结束曲抽取完成 -> 毕业典礼播放结束曲 -> 像素画毕业放映 -> 结局结算。
- 学年 BGM 为每学年 `3` 首，可切换；毕业结束曲在毕业典礼开始时播放，玩家不能切换。
- 评图等级统一为 `S / A / B / C / D / F`，GPA 上限统一为 `4.0`。
- 作品集改为总分制：最终评级 `D` 及以上课题的 `作品分` 累加，不设固定上限，不做轻衰减。
- 竞赛改为“图纸等级只做门槛，获奖概率读取作品分，S 必定一等奖”；奖金、简历记录和作品集分开结算。
- 万里路统一为每次 `2` 次行动机会 + `2500` 金钱，收益明显高于普通事件，但不直接进入作品集或作品分。
- 属性成长不设学期硬上限，改用低初始值、轻衰减、状态代价、金钱代价和复合门槛控制成长曲线。
- 大四只提醒并允许设置路线意向；大五完成报名、申请、投递、考试或确认后，才形成路线正式参与记录。
- 导师不再影响汇报策略；导师只影响阶段任务、事件倾向和路线倾向。
