# 决策日志

本文只保留仍有追溯价值的历史决策摘要。现行规则以 `PRD.md -> systems.md -> numbers.md -> 内容文档` 为准；本文件不作为规则源。

## 2026-05-16 系统收敛

- 首版主循环确定为 `5` 年、`60` 周、每周 `3` 次行动、每学期 `1` 个课题与 `1` 次评图。
- 后期路线改为互斥读取：每局最终只读取 `1` 个主结局。
- 开发底座确定为统一 `GameState`、显式状态机、统一结算器、固定随机种子和可复现结算日志。
- 商店收束为轻量修正系统，不做库存、背包、耐久或 build 系统。

## 2026-05-18 事件节奏

- 单局随机事件目标调整为 `90-95` 次，标准目标按 `92` 次校准。
- 固定流程事件和暑假写生节点不计入随机事件目标。
- 事件池收束为普通随机事件、交互事件、模型周事件、万里路事件。
- AI、导师、课程、地点只作为事件语境标签，不新增独立事件池。

## 2026-05-21 数值与路线收敛

- 评图等级统一为 `S / A / B / C / D / F`，GPA 上限统一为 `4.0`。
- 作品集改为总分制：最终评级 `D` 及以上课题的 `作品分` 累加，不设固定上限，不做轻衰减。
- 竞赛改为“图纸等级只做门槛，获奖概率读取作品分”；奖金、简历记录和作品集分开结算。
- 大四只提醒并允许设置路线意向；大五完成报名、申请、投递、考试或确认后，才形成路线正式参与记录。
- 导师不再影响汇报策略；导师只影响阶段任务和事件语境。

## 2026-05-23 内容与开发前收敛

- 成长成就改为只首次触发、首次计分；未获得成就在图鉴中置灰且不展示达成条件。
- 后期正式参与路线后关闭或置灰其他路线入口；建筑工作和转行投递都只能确认 `1` 个具体目标。
- 状态收尾只保留 `普通 / 沉重`，删除 `昂扬毕业`。
- 人生结局标题按新版命名收敛，并新增 `保研失败`。
- 大四焦虑、大五感动和部分交互事件补充阶段限制，避免前期触发明显后期内容。
- 商店、校外兼职、接设计外包、角色被动与专属技能按新版表收敛。
- 开局新增玩家姓名与大学名称填写；该档案只用于存档、称呼和剧情文本，不参与数值。
- 大一下、大二下暑假写生改为固定顺序交互事件。

## 2026-05-24 文档源收敛

- `PRD.md` 合并开发前冻结口径；`development-freeze.md` 改为旧引用入口。
- `content-plan.md` 删除重复后期路线规则，只保留院校、岗位、课程和 UI 文案内容池。
- 导师阶段任务表的判定条件、奖励和处罚随导师文本维护在 `content-plan.md`，并由 `numbers.md` 引用。
- 外包/兼职收束为每周最多 `2` 次；星巴克会员周卡压力代价提高为 `+7`；基础周恢复调整为 `精力 +10 / 压力 -5`。
- `健身房年卡` 收束为本学年精力临时上限 `105`，并把每次 `健身运动` 的精力消耗改为 `-1`。
- 大学课程明确提供实际属性收益，在课程题结算时同时生效。
- 缺失数值的普通随机事件已补齐结算，避免事件正文存在但配置无法落地。
- 保研结局读取收束为：只有 `本校 / 211` 保研读取 `留校读研——专教续费成功`。
- `毕业失败——延毕` 的触发条件统一为未完成毕业设计。

## 2026-05-25 文档瘦身

- `unresolved-issues.md` 改为真正的问题清单；已确认口径不再放在那里重复维护。
- `README.md` 增加维护规则，明确不改已拍板的事件标题、结局标题、属性名、路线名和正文内容。
- 旧 PRD、旧工作日志和阶段拷问结论的有效内容已经合并进核心文档；若历史文档与核心文档冲突，一律以核心文档为准。
- `被迫停学——好好休息，同学` 保留“后来毕业”的后日谈，但规则上仍按本局中途失败结束；人生结局分数区间同步收敛为 `100-1000`。
