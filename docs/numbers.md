## 状态收益修正

重复选择同一行动不做固定衰减，收益由精力和压力决定。

| 状态 | 条件 | 影响 |
|---|---|---|
| 状态良好 | 精力 >= 60 且压力 <= 50 | 正常收益 |
| 状态波动 | 精力 35-59 或压力 51-75 | 正向收益 -10% |
| 状态危险 | 精力 20-34 或压力 76-85 | 正向收益 -20%，给提醒 |
| 崩溃边缘 | 精力 < 20 或压力 > 85 | 正向收益 -35%，事件风险提高 |

提醒只保留三类：

- 精力见底
- 压力过载
- 状态危险

## 个人属性加成

个人属性会反向影响对应行动收益。

| 属性值 | 加成 |
|---|---|
| 0-39 | 对应收益 -10% |
| 40-59 | 正常 |
| 60-79 | 对应收益 +8% |
| 80-100 | 对应收益 +15% |

属性成长软上限：

- 六项角色属性达到 `80` 后，来自行动、随机事件和商店的正向属性增量减半，向下取整
- 单次原始增量为 `+1` 时，若该属性已经 `>= 80`，本次不再增加
- 同一学期内，同一项角色属性通过行动、随机事件和商店累计最多增加 `8`
- 导师阶段任务、万里路阶段奖励、角色专属技能不受软上限影响

## 随机事件数值结算

事件文本保留在 `events.md`，数值结算统一收口在这里。

收束原则：

- 每条事件单独对应一条数值结算
- 单个事件默认只改 `1-4` 个字段
- 事件默认不直接修改 `GPA / 作品集 / 简历价值 / 竞赛结果`
- 事件导致金钱降到 `0` 时，仍按总规则立即触发 `生活费断裂`
- 所有数值结算后统一做边界截断：`精力 / 压力 / 进度 / 作品质量 / 作品集 / 六项角色属性` 保持在 `0-100`

### 程序落地格式

首版程序层只需要维护以下字段：

```text
event_id
title
pool
result
option_a_result
option_b_result
note
```

字段说明：

- `pool` 只允许：`normal / interactive / model / travel`
- 普通事件、模型周事件、万里路事件只填 `result`
- 交互事件只填 `option_a_result / option_b_result`
- `result` 建议直接写成结构化键值，例如 `{"progress":-6,"pressure":6}`
- `title` 不保证唯一，当前至少有 `老师让你重画` 重复，程序主键必须是 `event_id`

```text
评图总分 = 进度 40% + 作品质量 35% + 汇报策略结果 25% + 导师任务修正
```

等级与绩点：

- A：`4.0`，总分 `>= 85`
- B：`3.3`，总分 `75-84`
- C：`2.7`，总分 `60-74`
- D：`2.0`，总分 `45-59`
- F：`0.0`，总分 `< 45`

规则：

- `D` 视为过关
- `F` 视为挂科
- 连续 `2` 次 `F` 直接触发 `连续挂科退学`

## GPA 结算

- 开局显示 `GPA = 4.0`
- 每学期先根据评图等级得到设计课绩点
- 再叠加大学课程修正
- 得到该学期最终绩点后，再并入个人累计 GPA

累计规则：

```text
个人GPA = 所有已完成学期最终绩点的平均值
```

## 汇报策略

| 策略 | 门槛 | 基础汇报段得分 | 规则 |
|---|---|---:|---|
| 我就是大师 | 设计水平 >= 70 且汇报表达 >= 60 | 88 | 不达门槛降为 62 |
| 高级技术流 | 软件技术 >= 75 且创意审美 >= 65 | 86 | 缺任一门槛降为 60 |
| 求求你别挂我 | 压力 >= 75 | 72 | 不达门槛降为 52 |
| 直接念 PPT | 无门槛 | 60 | 稳定保底 |

状态修正：

- 精力 < 30：汇报段 -8
- 压力 > 80：除“求求你别挂我”外，其余策略 -5
- 精力 = 0：额外 -5，但不直接失败

导师偏好原则：

- 每位导师只有 `1` 个明显偏好策略
- 每位导师只有 `1` 个明显厌恶策略
- 其余策略视为中性
- 导师偏好只作用在汇报段得分
- 同一导师策略最大分差不超过 `12`

## 大学课程修正

- 每学期结束回答 `3` 道对应课程题
- 答对 `2` 题：该学期 GPA 修正 `+0.2`
- 答对 `1` 题：该学期 GPA 修正 `+0.1`
- 答对 `0` 题：该学期 GPA 修正 `-0.2`
- 答错时直接显示正确答案

## 强路线稀有度

强路线不是失败条件，只影响成功通关时的主结局优先级。

| 强度 | 路线 | 目标体感 |
|---|---|---|
| 顶级强路线 | 保研直升、顶级名校留学、选调回乡、大师建筑事务所 | 需要明确规划，普通游玩不稳定达成 |
| 中强路线 | 考研上岸、普通留学、国企设计院、AI 新职业 | 合理规划可达成 |
| 普通路线 | 地方设计院、普通事务所、非 AI 转行、普通但稳定 | 大多数未失败玩家可回落 |

调参原则：

- 若强路线过多触发，优先上调 `作品集 / GPA / 实习价值 / AI 相关经历` 门槛
- 若普通通关过难，优先降低失败压力，不降低强路线门槛
- `普通但稳定` 不读取作品集、竞赛、实习等强路线条件

- 被迫停学：精力 `= 0` 且压力 `> 90`，并在周末结算时仍未脱离
