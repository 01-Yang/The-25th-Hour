# 数值规则

## 值域

- 精力：基础上限 `100`；获得 `健身房年卡` 的学年内，临时上限为 `105`
- 压力：`0-100`
- 进度：普通学期 `0-100`，大五毕业设计 `0-200`
- 作品质量：普通学期 `0-100`，大五毕业设计 `0-200`
- 作品集总分：不设固定上限，首版按约 `0-900` 的体感区间校准
- 六项角色属性：`0-100`
- GPA：`0.0-4.0`
- 金钱：人民币元

补充规则：

- 任意结算后若金钱小于 `0`，立即触发中途失败结局 `生活费断裂`；金钱等于 `0` 不触发失败。
- 所有角色初始精力固定为 `100`；角色开局差异主要来自初始压力、初始金钱、六项角色属性、被动特质和专属技能。
- 临时精力上限只改变精力截断上限，不改变行动次数、周恢复规则或 `被迫停学` 判定。

## 结算与实现口径

- 所有数值变化必须写成结构化 delta，并交由统一结算器处理。
- 结算器统一负责上下限、六项属性轻衰减、失败检查、隐藏结果缓存和结算日志。
- 行动、事件、商店、实习、评图、考试、路线和结局不得各自绕过结算器直接改状态。
- 每局生成固定随机种子；事件抽取、考试抽题、实习录取、留学申请和岗位投递内部判定都必须可复现。
- 程序主键使用稳定 ID，展示标题和文案不作为规则读取依据。
- 规则触发必须读取明确状态字段、阈值、次数、周次、阶段、布尔标记或表格概率；不能只读取“较高 / 较低 / 明显 / 一般 / 倾向”等描述性文字。

## 角色初始六维

初始六维采用低起点设计，避免大一、大二过快满级。普通角色大多数属性落在 `20-34`，明显强项落在 `35-45`，首版不设置任何初始属性高于 `45`。

| 普通人 | 28 | 26 | 28 | 26 | 28 | 28 |
| 混得入 | 22 | 24 | 24 | 22 | 32 | 26 |

成长节奏目标：

- 大一：多数属性仍在 `20-45`，只让玩家感到方向差异，不让强项成型过快。
- 大二：专注培养的属性进入 `45-65`，开始满足普通实习与部分低门槛路线前置。
- 大三：核心属性进入 `60-85`，角色 build 感开始明显。
- 大四下：长期专注的 `1-3` 项属性可以接近或达到 `100`。
- 大五：不再主要承担基础属性养成，而是读取前四年形成的属性、作品集、实习和路线结果。

## 行动联动修正

### 画图与赶图

- `正常速度画图` 与 `爆肝通宵赶图` 只直接负责 `进度` 推进，不直接增加 `作品质量`。
- 所有带 `基础进度` 的常驻行动都读取 `软件技术` 档位修正；事件进度加减不读取该修正。
- 普通学期进度硬门槛为 `100`；正常速度画图的调参目标是 `6-7` 次行动补满进度。
- 爆肝通宵赶图是高代价捷径，调参目标是 `3-4` 次行动补满进度。
- 随机事件、交互事件、导师任务和专属技能可以增减进度，因此实际所需画图次数允许上下浮动 `1-2` 次。
- 若玩家连续遇到负向进度事件，系统不额外放宽进度硬门槛，玩家需要通过画图、爆肝或技能补救。

| 软件技术 | 常驻进度行动修正 |
|---|---|
| `0-39` | 无修正 |
| `40-59` | 进度 `+1` |
| `60-79` | 进度 `+2` |
| `80-100` | 进度 `+3` |

- 无论 `软件技术` 多高，`爆肝通宵赶图` 的单次压力增量最低仍为 `+8`，单次精力消耗最低仍为 `-14`。
- 软件技术不再给低档惩罚，只提供小幅正向加成，避免大一阶段因为初始属性低而被进度硬门槛双重惩罚。

### 质量类行动

- 直接增加 `作品质量` 的常驻行动只读取 `方案推敲` 与 `场地调研`。
- `阅读/展览/讲座` 主要负责累计 `创意审美`，不直接增加 `作品质量`。
- `方案推敲` 与 `场地调研` 主要增加 `作品质量`，并固定附带 `进度 +1`，代表方案深化时顺手补完图纸逻辑。
- 所有带 `基础作品质量` 的常驻行动都读取 `设计水平 + 创意审美` 档位修正；事件、模型周材料和专属技能按各自表格结算，不额外读取该修正。
- 质量修正采用简单相加口径：先计算 `质量理解 = floor((设计水平 + 创意审美) / 2)`，再按下表修正行动的 `基础作品质量`：

| 质量理解 | 质量行动修正 |
|---|---|
| `< 40` | `0` |
| `40-59` | `+1` |
| `60-79` | `+2` |
| `80-100` | `+3` |

- 质量类行动不再设置低属性惩罚；低属性只是不加成，避免大一阶段质量成长压力过高。
- 普通学期按 `18-20` 次可用行动校准：调参目标是 `4-5` 次质量投入可稳定摸到 `D` 门槛，`6-7` 次质量投入可进入 `C / B` 区间。
- 进度与质量合计不应吃掉整个学期；普通过关路线的调参目标是留下 `7-10` 次行动给学习、运动、放松、社交、兼职、万里路或突发补救。

## 状态收益修正

重复选择同一行动不做固定衰减，收益由精力和压力决定。

| 状态 | 条件 | 影响 |
|---|---|---|
| 正常 | 精力 >= `60` 且压力 <= `50` | 正常收益 |
| 轻度风险 | 精力 >= `30` 且 < `60`，或压力 > `50` 且 <= `80` | 行动产生的正向 `进度 / 作品质量 / 六项属性` 各自 `-1`，最低仍保留 `+1`；UI 启用浅红色低频呼吸 |
| 高危 | 精力 < `30` 或压力 > `80` | 行动产生的正向 `进度 / 作品质量 / 六项属性` 各自 `-2`，最低仍保留 `+1`；事件风险提高；UI 启用强红色呼吸并触发对应行动限制 |

提醒只保留两类：

- 状态高危
- 金钱高危

高危行动限制：

| 高危类型 | 条件 | 限制 |
|---|---|---|
| 精力高危 | 精力 < `30` | 禁用高消耗行动或持续消耗入口：`爆肝通宵赶图 / 接设计外包 / 校外兼职 / 万里路 / 实习申请`；交互事件中高消耗选项不额外警告，若结算后精力小于 `0` 则直接触发 `被迫停学` |
| 压力高危 | 压力 > `80` | 禁用会增加压力的常驻行动、专属技能和交互事件选项；负向事件基础权重 +20 |
| 双高危 | 精力 < `30` 且压力 > `80` | 只保留 `休养生息`、当前可购买且不会增加对应高危资源风险的商店恢复商品和必要结算操作 |

高危提示文案口径：

- 状态高危：`你正在崩溃边缘，别画图了，快去休息！`
- 金钱高危：`你快要饿死了，省着点花！`
- 金钱高危触发条件：当前金钱低于该角色家庭背景对应的 `1` 周自动花费。

### 状态 UI 呼吸

状态 UI 只做轻量视觉提示，不新增可交互系统。

| 状态 | 条件 | UI 呼吸规则 |
|---|---|---|
| 正常 | 精力 >= `60` 且压力 <= `50` | 不启用呼吸，仅显示普通状态条 |
| 轻度风险 | 精力 >= `30` 且 < `60`，或压力 > `50` 且 <= `80` | 状态区启用浅红色低频呼吸，提示玩家需要调整节奏 |
| 高危 | 精力 < `30` 或压力 > `80` | 状态区启用强红色中高频呼吸，对应高消耗或高压行动置灰 |

- 呼吸效果只改变颜色、透明度和轻微亮度，不改变布局尺寸，不遮挡按钮文字。
- 呼吸状态以最严重的一档为准。
- 当精力和压力同时触发不同档位时，按更严重档位显示；同档位同时触发时，仍只显示一次红色呼吸，不叠加动画。
- 浅红色呼吸触发条件为：`精力 >= 30 且 < 60`，或 `压力 > 50 且 <= 80`。
- 强红色呼吸与对应行动限制触发条件为：`精力 < 30` 或 `压力 > 80`。
- 高压和低精力统一使用红色呼吸，不使用蓝色或双色风险提示。
- 进入评图、结局和固定剧情文本时，呼吸效果降弱，避免干扰阅读。

## 每周自动恢复

- 每周所有行动、随机事件、交互事件、商店购买、实习收益/代价、自动生活费结算完成后，统一执行基础周恢复：`精力 +10`、`压力 -5`
- 周恢复不占行动次数，不因 `模型周 / 赶图周 / 考试周` 改变
- 周恢复后再判断是否仍处于高危状态；若仍处于高危，只保留高危限制，不追加额外保底恢复

## 属性作用与成长

六项角色属性不再提供全局百分比加成，避免和行动档位修正重复叠加。每个属性只影响明确系统、行动档位或路线门槛：

- `软件技术`：影响 `正常速度画图 / 爆肝通宵赶图` 的进度档位修正，并参与技术型路线、实习和竞赛公式。
- `设计水平 + 创意审美`：共同影响 `方案推敲 / 场地调研` 的质量档位修正，并参与作品集、竞赛和多数建筑路线门槛。
- `汇报表达`：影响汇报策略可选门槛、体制内/升学/竞赛等表达型门槛。
- `人际交往`：影响社交、实习、体制内路线和部分事件选项。
- `抗压能力`：影响高压路线门槛、体制内路线和少数抗压相关事件。

属性成长规则：

- 不设置同一学期属性成长硬上限；玩家选择的行动、事件、商店、课程和实习只要完成，就正常结算，不因为本学期已成长过多而吞掉收益。
- 六项角色属性仍统一截断在 `0-100`；达到 `100` 后可以继续执行相关行动，但属性不再超过上限。
- 所有来源的六项属性变化都统一进入轻衰减，包括行动、事件、商店、课程、实习、导师阶段任务、万里路和角色技能。
- 当某项属性当前值低于 `80` 时，正向属性成长按表格原值结算。
- 当某项属性当前值达到 `80` 及以上时，正向属性成长进入轻衰减：最终成长 = `ceil(原成长 / 2)`，且只要原成长为正，最终至少为 `+1`。
- 当某项属性当前值高于 `20` 时，负向属性下降按表格原值结算。
- 当某项属性当前值为 `20` 及以下时，负向属性下降进入轻衰减：最终下降 = `ceil(原下降绝对值 / 2)`，且只要原下降为负，最终至少为 `-1`。
- `进度 / 作品质量 / 作品分 / 作品集总分 / 精力 / 压力 / 金钱 / GPA` 不使用六项属性轻衰减，按各自系统规则结算。
- 属性成长控制优先通过低初始值、较小单次增量、状态代价、金钱代价、路线复合门槛和轻衰减完成，不用学期硬上限控制。

## 随机事件数值结算

事件文本保留在 `events.md`，数值结算统一收口在这里。

收束原则：

- 每条事件单独对应一条数值结算
- 单个事件默认只改 `1-4` 个字段
- 事件默认不直接修改 `GPA / 作品集 / 简历价值 / 竞赛结果`
- 事件导致金钱小于 `0` 时，仍按总规则立即触发 `生活费断裂`
- 所有数值结算后统一做边界截断：`压力 / 六项角色属性` 保持在 `0-100`；普通学期的 `进度 / 作品质量 / 作品分` 保持在 `0-100`，大五毕业设计的 `进度 / 作品质量` 保持在 `0-200`；作品集总分不设固定上限；精力允许先结算到负数并立即触发 `被迫停学`

## 评图

评图采用三段制：先判定进度硬门槛，再由作品质量得到质量评图分与基础图纸评级，最后结算汇报策略对最终等级、作品分和设计课绩点的影响。汇报策略完成后先显示“汇报成功 / 汇报失败”，再进入最终评图等级展示。

进度硬门槛：

| 阶段 | 进度要求 | 未达成结果 |
|---|---:|---|
| 大一至大四普通课题 | 评图前进度必须达到 `100 / 100` | 本次评图直接为 `F`，不可用汇报策略补救 |
| 大五上毕业设计期中 | 期中汇报前毕业设计进度必须达到 `100 / 200` | 本次评图直接为 `F`，不可用汇报策略补救 |
| 大五下毕业设计答辩 | 答辩前毕业设计进度必须达到 `200 / 200` | 未完成毕业设计，读取 `毕业失败——延毕` |

质量评图分：

- 大一至大四直接读取当学期作品质量，范围 `0-100`。
- 大五毕业设计作品质量范围为 `0-200`；评图时折算为 `质量评图分 = floor(作品质量 / 2)`。
- 若质量评图分低于 `45`，基础图纸评级为 `F`。
- 质量不足导致的 `F` 可以选择 `求求你别挂我` 尝试补救；进度不足导致的 `F` 不进入可补救分支。
- 基础图纸评级先由质量评图分决定；随后汇报策略可以在固定成功率下改变最终图纸评级、作品分和设计课绩点。

导师阶段任务、模型材料、事件和角色技能只允许在评图前改变 `作品质量 / 进度 / 状态 / GPA 学期修正`。汇报策略不再改变 `进度` 或 `作品质量`，只在基础图纸评级产生后影响 `最终图纸评级 / 作品分 / 设计课绩点`。

导师阶段任务数值口径：

- 每学期第 `1` 周发布当前导师的 `1` 个阶段任务。
- 阶段任务只统计当学期行动、实习、竞赛投稿、当学期课题状态和本学期属性成长量；新学期开始时任务进度清零。
- 阶段任务统一在评图前结算，只有 `完成 / 未完成` 两档，不做半完成。
- 完成奖励和未完成处罚以 `content-plan.md` 的导师阶段任务表为准。
- 完成奖励允许提高 `作品质量 / 进度 / 六项角色属性 / GPA 学期修正 / 状态`，但不直接改变最终评图等级。
- 未完成处罚允许降低 `作品质量 / GPA 学期修正` 或提高压力；不直接触发失败结局。

作品分：

- `作品分` 是这份课题在评图和汇报后的最终量化分数，用于竞赛获奖概率、作品集增长、路线门槛和结局结算。
- 基础作品分默认等于 `质量评图分`。
- 汇报策略可以在基础作品分上加减固定分值，但最终仍截断在 `0-100`。
- 最终作品分必须落在最终图纸评级对应的分数区间内：`S=95-100`，`A=85-94`，`B=75-84`，`C=60-74`，`D=45-59`，`F=0-44`。
- 若汇报策略导致评级升档或降档，先调整最终图纸评级，再把作品分截断到新评级区间内；例如 `B` 升到 `A` 后，最终作品分最低为 `85`。
- 进度硬门槛未达成导致的 `F`：最终作品分固定为 `0`，不进入作品集。
- 质量不足导致的 `F`：未补救时最终作品分保持在 `0-44`；若 `求求你别挂我` 成功补救为 `D`，最终作品分至少为 `45`，最高仍不超过 `59`。

等级与绩点：

| 基础图纸评级 | 设计课绩点 | 质量评图分 |
|---|---:|---|
| S | `4.0` | `95-100` |
| A | `3.7` | `85-94` |
| B | `3.2` | `75-84` |
| C | `2.4` | `60-74` |
| D | `1.6` | `45-59` |
| F | `0.0` | `<45` 或进度硬门槛未达成 |

规则：

- `D` 视为过关
- `F` 视为挂科
- 每次评图结算完成后，立刻向玩家显示本次最终图纸评级与作品分。
- 连续 `2` 次 `F` 直接触发 `连续挂科退学`

## GPA 结算

- 开局显示 `GPA = 4.0`
- 每学期先根据评图等级得到设计课绩点
- 再叠加本学期 `GPA学期修正`
- 得到该学期最终绩点后，再并入个人累计 GPA
- 单学期最终绩点截断在 `0.0-4.0`
- GPA 修正使用固定加减值，不使用百分比。
- 评图等级是 GPA 的主要来源；`GPA学期修正` 只做小幅修正，不能把 `F` 补成合格绩点。
- 若本学期最终图纸评级为 `F`，正向 `GPA学期修正` 不生效；负向 `GPA学期修正` 仍生效，但最终绩点最低截断为 `0.0`。

累计规则：

```text
个人GPA = 所有已完成学期最终绩点的平均值
```

示例区分：

- `S + GPA学期修正 +0.2`：`4.0`，因为已到上限。
- `A + GPA学期修正 +0.2`：`3.9`。
- `B + GPA学期修正 +0.1`：`3.3`。
- `C + GPA学期修正 -0.2`：`2.2`。
- `D + GPA学期修正 -0.2`：`1.4`。
- `F`：`0.0`。

汇报策略通用规则：

## 作品集

作品集采用总分制，不再是 `0-100` 的单项属性。

规则：

- `作品集总分 = 所有已完成且最终图纸评级为 D 及以上课题的作品分之和`。
- `F` 评级作品不进入作品集，作品集增量为 `0`。
- 作品集总分不设固定上限，不做轻衰减；首版按约 `0-900` 的体感区间校准。
- 作品集只保留一个总分，不拆子分。
- 竞赛不直接增加作品集总分；竞赛只记录投稿、奖项、奖金和简历标签。
- 万里路不进入作品集总分，只进入简历与成长路径。
- 实习、商店、课程、随机事件和导师阶段任务都不能直接增加作品集总分；它们只能通过影响作品质量、最终作品分或简历标签间接发挥作用。
- 大一至大四每次正式评图结算后，若最终评级为 `D` 及以上，本次作品分立即入库。
- 大五毕业设计只在大五下最终答辩后入库 `1` 次；大五上期中汇报只做阶段评图和风险反馈，不把同一毕业设计重复计入作品集。

## 建筑生的万里路

触发规则：

- 万里路按固定地点顺序推进；玩家选择前往当前地点后，必定触发对应万里路事件。
- 每次消耗 `2` 次行动机会，不再消耗整周全部行动。
- 每次额外消耗金钱 `2500`；余额不足时入口置灰。

## 升学路线组

升学路线必须在大五阶段正式参与 `保研 / 考研 / 留学` 之一。保研和考研不能同时进行；大四阶段只允许设置可修改的路线意向。

### 升学专业考试

保研复试与考研初试共用 `升学专业题库`。

- 题库总量：`40` 道单选题
- 每次抽取：`10` 道
- 难度组成：`6` 道简单题 + `4` 道困难题
- 内部先记录答对题数，再折算为现实分数线
- 答题过程中不反馈单题对错，也不显示正确答案
- 大五上评图结束后，若已正式参与 `保研` 或 `考研` 且达到对应申请门槛，则强制进入本路线考试阶段
- 考试结束后只记录内部得分与是否过线，不向玩家公开分数、答对题数或上岸结果
- 考试结束后弹出统一等待结果提示；文案读取 `content-plan.md` 的“路线等待结果弹窗文案”。
- 保研、考研是否成功统一延后到大五下毕业设计答辩后的结局结算再揭示
- 折算规则：答对 `10 / 9 / 8 / 7 / 6` 题时，分别折算为 `410 / 395 / 380 / 365 / 350` 分；答对 `5` 题及以下统一视为 `350` 分以下

### 行测考试

- 题库总量：`30` 道单选题
- 每次抽取：`10` 道
- 难度组成：`6` 道简单题 + `4` 道困难题
- 内部先记录答对题数，再折算为现实分数线
- 答题过程中不反馈单题对错，也不显示正确答案
- 大五上评图结束后，若已正式参与 `选调 / 考公 / 考编` 且达到对应报名门槛，则强制进入行测考试阶段
- 行测结束后只记录内部得分与是否过线，不向玩家公开分数、答对题数或上岸结果
- 行测结束后弹出统一等待结果提示；文案读取 `content-plan.md` 的“路线等待结果弹窗文案”。
- 选调、考公、考编是否成功统一延后到大五下毕业设计答辩后的结局结算再揭示
- 折算规则：答对 `10 / 9 / 8 / 7 / 6` 题时，分别折算为 `150 / 145 / 140 / 135 / 130` 分；答对 `5` 题及以下统一视为 `130` 分以下

## 毕业设计完成门槛

顺利毕业必须完成毕业设计；否则无论路线是否达标，都读取 `毕业失败——延毕`。

大五上和大五下共同承载同一个毕业设计，进度与作品质量不在大五上结束后清零。大五毕业设计的进度与作品质量上限均为 `200`。

大五上期中汇报条件：

- 大五上毕业设计期中汇报前，毕业设计进度必须达到 `100 / 200`；未达成则本次评图直接为 `F`
- 大五上毕业设计期中汇报的作品质量按梯度判定，至少达到不被评为 `F` 的门槛

毕业设计完成条件：

- 大五下毕业设计评图等级为 `D` 或以上
- 大五下评图前毕业设计进度必须达到 `200 / 200`
- 大五下毕业设计作品质量折算后的质量评图分必须达到 `D` 或以上
- 大五下毕业设计答辩一结束，立即进入结局系统结算；此前所有路线考试、申请、投递的内部结果都在此时统一读取

## 强路线稀有度

强路线不是失败条件，只影响成功通关时的主结局优先级。

| 强度 | 路线 | 目标体感 |
|---|---|---|
| 顶级强路线 | 强校上岸、顶级名校留学、选调回乡、大师建筑事务所 | 需要明确规划，普通游玩不稳定达成 |
| 中强路线 | 考研上岸、普通留学、国企设计院、AI 新职业 | 合理规划可达成 |
