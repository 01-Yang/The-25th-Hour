# 数值规则

## 值域

- 精力：`0-100`
- 压力：`0-100`
- 进度：普通学期 `0-100`，大五毕业设计 `0-200`
- 作品质量：普通学期 `0-100`，大五毕业设计 `0-200`
- 作品集：`0-100`
- 六项角色属性：`0-100`
- GPA：`0.0-4.0`
- 金钱：人民币元

补充规则：

- 任意结算后若金钱小于 `0`，立即触发中途失败结局 `生活费断裂`；金钱等于 `0` 不触发失败。
- 所有角色初始精力固定为 `100`；角色开局差异主要来自初始压力、初始金钱、六项角色属性、被动特质和专属技能。

## 结算与实现口径

- 所有数值变化必须写成结构化 delta，并交由统一结算器处理。
- 结算器统一负责上下限、学期成长上限、失败检查、隐藏结果缓存和结算日志。
- 行动、事件、商店、实习、评图、考试、路线和结局不得各自绕过结算器直接改状态。
- 每局生成固定随机种子；事件抽取、考试抽题、实习录取、留学申请和投递结果都必须可复现。
- 程序主键使用稳定 ID，展示标题和文案不作为规则读取依据。

## 角色初始六维

初始六维采用低起点设计，避免大一、大二过快满级。普通角色大多数属性落在 `20-34`，明显强项落在 `35-45`，首版不设置任何初始属性高于 `45`。

成长节奏目标：

- 大一：多数属性仍在 `20-45`，只让玩家感到方向差异，不让强项成型过快。

- 大四下：长期专注的 `1-3` 项属性可以接近或达到 `100`。
- 大五：不再主要承担基础属性养成，而是读取前四年形成的属性、作品集、实习和路线结果。

## 行动联动修正

### 画图与赶图

- `正常速度画图` 与 `爆肝通宵赶图` 只直接负责 `进度` 推进，不直接增加 `作品质量`。
- 所有带 `基础进度` 的常驻行动都读取 `软件技术` 档位修正；事件进度加减不读取该修正。
- 普通学期进度硬门槛为 `100`；正常速度画图的目标压力是约 `6-7` 次行动补满进度。
- 爆肝通宵赶图是高代价捷径，目标压力是约 `3-4` 次行动补满进度。
- 随机事件、交互事件、导师任务和专属技能可以增减进度，因此实际所需画图次数允许上下浮动 `1-2` 次。
- 若玩家连续遇到负向进度事件，系统不额外放宽进度硬门槛，玩家需要通过画图、爆肝或技能补救。

| 软件技术 | 常驻进度行动修正 |
|---|---|
| `0-39` | 无修正 |
| `40-69` | 进度 `+1` |
| `70-100` | 进度 `+2` |

- 无论 `软件技术` 多高，`爆肝通宵赶图` 的单次压力增量最低仍为 `+8`，单次精力消耗最低仍为 `-14`。
- 软件技术不再给低档惩罚，只提供小幅正向加成，避免大一阶段因为初始属性低而被进度硬门槛双重惩罚。

### 质量类行动

- 直接增加 `作品质量` 的常驻行动只读取 `方案推敲` 与 `场地调研`。
- `阅读/展览/讲座` 主要负责累计 `创意审美`，不直接增加 `作品质量`。
- `方案推敲` 与 `场地调研` 主要增加 `作品质量`，并固定附带 `进度 +2`，代表方案深化时顺手补完图纸逻辑。
- 所有带 `基础作品质量` 的常驻行动都读取 `设计水平 + 创意审美` 档位修正；事件、模型周材料和专属技能按各自表格结算，不额外读取该修正。
- 质量修正采用简单相加口径：先计算 `质量理解 = floor((设计水平 + 创意审美) / 2)`，再按下表修正行动的 `基础作品质量`：

| 质量理解 | 质量行动修正 |
|---|---|
| `< 40` | `0` |
| `40-59` | `+1` |
| `60-79` | `+2` |
| `80-100` | `+3` |

- 质量类行动不再设置低属性惩罚；低属性只是不加成，避免大一阶段质量成长压力过高。
- 普通学期按 `18-20` 次可用行动校准：约 `4-5` 次质量投入可稳定摸到 `D` 的门槛，约 `6-7` 次质量投入可进入 `C / B` 区间，`A` 需要属性加成、模型周、导师任务或正向事件配合。
- 进度与质量合计不应吃掉整个学期；普通过关路线应能留下约 `7-10` 次行动给学习、运动、放松、社交、兼职、万里路或突发补救。

## 状态收益修正

重复选择同一行动不做固定衰减，收益由精力和压力决定。

| 状态 | 条件 | 影响 |
|---|---|---|
| 状态良好 | 精力 >= `60` 且压力 <= `50` | 正常收益 |
| 状态波动 | 精力 `45-59` 或压力 `51-60` | 正向收益 -10% |
| 状态危险 | 精力 `20-44` 或压力 `61-80` | 正向收益 -20%，给提醒；UI 启用浅红色呼吸 |
| 崩溃边缘 | 精力 < `20` 或压力 > `80` | 正向收益 -35%，事件风险提高；UI 启用强红色呼吸并触发对应行动限制 |

提醒只保留三类：

- 精力见底
- 压力过载
- 状态危险

高危行动限制：

| 高危类型 | 条件 | 限制 |
|---|---|---|
| 精力高危 | 精力 < `20` | 禁用高消耗行动：`爆肝通宵赶图 / 接设计外包 / 校外兼职 / 万里路 / 实习开始`；交互事件中高消耗选项不额外警告，若结算后精力小于 `0` 则直接触发 `被迫停学` |
| 压力高危 | 压力 > `80` | 禁用高压行动：`爆肝通宵赶图 / 接设计外包 / 高压专属技能 / 高压交互选项`；事件负向权重 +20% |
| 双高危 | 精力 < `20` 且压力 > `80` | 只保留恢复类行动、低风险商店补给和必要结算操作 |

高危提示文案口径：

- 精力高危：`你已经快撑不住了，先恢复精力，否则很多行动无法继续。`
- 压力高危：`压力已经接近失控，继续硬扛会锁住高压行动。`
- 双高危：`当前状态非常危险，本周必须先把自己拉回来。`

### 状态 UI 呼吸

状态 UI 只做轻量视觉提示，不新增可交互系统。

| 状态 | 条件 | UI 呼吸规则 |
|---|---|---|
| 正常 | 精力 >= `45` 且压力 <= `60` | 不启用呼吸，仅显示普通状态条 |
| 轻度风险 | 精力 >= `20` 且 < `45`，或压力 > `60` 且 <= `80` | 状态区启用浅红色低频呼吸，提示玩家需要调整节奏 |
| 精力高危 | 精力 < `20`，且压力 <= `80` | 状态区启用强红色中频呼吸，高消耗行动置灰 |
| 压力高危 | 压力 > `80`，且精力 >= `20` | 状态区启用强红色中高频呼吸，高压行动置灰 |
| 双高危 | 精力 < `20` 且压力 > `80` | 界面边缘启用强红色中高频呼吸，只保留恢复和必要结算入口 |

- 呼吸效果只改变颜色、透明度和轻微亮度，不改变布局尺寸，不遮挡按钮文字。
- 呼吸状态以最严重的一档为准；双高危优先级最高。
- 当精力和压力同时触发不同档位时，按更严重档位显示；同档位同时触发时，仍只显示一次红色呼吸，不叠加动画。
- 浅红色呼吸触发条件为：`精力 < 45` 或 `压力 > 60`。
- 强红色呼吸与对应行动限制触发条件为：`精力 < 20` 或 `压力 > 80`。
- 高压和低精力统一使用红色呼吸，不使用蓝色或双色风险提示。
- 进入评图、结局和固定剧情文本时，呼吸效果降弱，避免干扰阅读。

## 每周自动恢复

- 每周所有行动、随机事件、交互事件、商店购买、实习收益/代价、自动生活费结算完成后，统一恢复：`精力 +12`、`压力 -10`
- 周恢复不占行动次数，不因 `模型周 / 赶图周 / 考试周` 改变
- 周恢复后再判断是否仍处于高危状态；若仍处于高危，只保留高危限制，不追加额外保底恢复

## 个人属性加成

个人属性会反向影响对应行动收益。

| 属性值 | 加成 |
|---|---|
| 0-39 | 对应收益 -10% |
| 40-59 | 正常 |
| 60-79 | 对应收益 +8% |
| 80-100 | 对应收益 +15% |

属性成长规则：

- 不设置 `80` 后成长衰减，属性增长速度始终按事件、行动和奖励表直接结算。
- 同一学期内，同一项角色属性通过行动、随机事件、交互事件、商店、课程和实习累计最多增加 `6`。
- 导师阶段任务、万里路阶段奖励、角色专属技能和结局前路线节点不受学期成长上限影响。
- 负向事件导致属性下降时不受成长上限影响，直接结算并截断在 `0-100`。
- 属性成长控制优先通过低初始值、较小单次增量、学期成长上限和负向事件完成，不新增复杂衰减机制。

## 随机事件数值结算

事件文本保留在 `events.md`，数值结算统一收口在这里。

收束原则：

- 每条事件单独对应一条数值结算
- 单个事件默认只改 `1-4` 个字段
- 事件默认不直接修改 `GPA / 作品集 / 简历价值 / 竞赛结果`
- 事件导致金钱小于 `0` 时，仍按总规则立即触发 `生活费断裂`
- 所有数值结算后统一做边界截断：`压力 / 作品集 / 六项角色属性` 保持在 `0-100`；普通学期的 `进度 / 作品质量` 保持在 `0-100`，大五毕业设计的 `进度 / 作品质量` 保持在 `0-200`；精力允许先结算到负数并立即触发 `被迫停学`

### 程序落地格式

程序层与规则层统一使用同一套事件字段口径：

```text
event_id
title
pool
phase
sentiment
tags
context_tags
base_weight
cooldown_weeks
repeatable
ai_related
ai_experience_delta
trigger_rules
result
options
note
```

字段说明：

- `pool` 只允许：`normal / interactive / model / travel`
- `phase` 用于限制事件只在对应阶段参与抽取
- `tags` 只服务筛选、冷却和权重，不新增玩法
- `context_tags` 只描述导师、课程、AI、地点等语境
- `repeatable` 与 `cooldown_weeks` 共同控制重复抽取
- `trigger_rules` 只描述保底、周次、阶段、黑名单等触发覆盖条件
- 普通事件、模型周事件、万里路事件写 `result`
- 交互事件写 `options`
- `result` 与 `options` 都建议直接写成结构化键值
- `title` 不作为程序主键，程序主键必须使用 `event_id`

首版最小必填字段：

```text
event_id
title
pool
phase
result / options
```

补充规则：

- 首版可以先只消费最小必填字段完成抽取与结算
- `tags / context_tags / base_weight / cooldown_weeks / repeatable / ai_related / ai_experience_delta / trigger_rules` 仍视为正式字段，不再另起第二套简化格式

## 评图

评图采用三段制：先判定进度与质量硬门槛，再由作品质量得到基础图纸评级与作业分，最后结算汇报策略对最终等级、作业分和设计课绩点的影响。汇报策略的成功或失败不单独展示，评图结束后直接显示最终图纸评级。

进度硬门槛：

| 阶段 | 进度要求 | 未达成结果 |
|---|---:|---|
| 大一至大四普通课题 | 评图前进度必须达到 `100 / 100` | 本次评图直接为 `F` |
| 大五上毕业设计期中 | 期中汇报前毕业设计进度必须达到 `100 / 200` | 本次评图直接为 `F` |
| 大五下毕业设计答辩 | 答辩前毕业设计进度必须达到 `200 / 200` | 未完成毕业设计，读取 `没毕业` |

质量评图分：

- 大一至大四直接读取当学期作品质量，范围 `0-100`。
- 大五毕业设计作品质量范围为 `0-200`；评图时折算为 `质量评图分 = floor(作品质量 / 2)`。
- 若质量评图分低于 `45`，基础图纸评级为 `F`。
- 基础图纸评级先由质量评图分决定；随后汇报策略可以在固定成功率下改变最终图纸评级、作业分和设计课绩点。

导师阶段任务、模型材料、事件和角色技能只允许在评图前改变 `作品质量 / 进度 / 状态 / GPA 学期修正`。汇报策略不再改变 `进度` 或 `作品质量`，只在基础图纸评级产生后影响 `最终图纸评级 / 作业分 / 设计课绩点`。

作业分：

- `作业分` 是这份作业的最终量化分数，用于后续对接竞赛、作品集筛选、路线门槛和结局结算。
- 基础作业分默认等于 `质量评图分`。
- 汇报策略可以在基础作业分上加减固定分值，但最终仍截断在 `0-100`。

等级与绩点：

| 基础图纸评级 | 设计课绩点 | 质量评图分 |
|---|---:|---|
| A | `4.0` | `85-100` |
| B | `3.3` | `75-84` |
| C | `2.7` | `60-74` |
| D | `2.0` | `45-59` |
| F | `0.0` | `<45` 或进度硬门槛未达成 |

规则：

- `D` 视为过关
- `F` 视为挂科
- 每次评图结算完成后，立刻向玩家显示本次最终图纸评级与作业分。
- 连续 `2` 次 `F` 直接触发 `连续挂科退学`

## GPA 结算

- 开局显示 `GPA = 4.0`
- 每学期先根据评图等级得到设计课绩点
- 再叠加大学课程修正
- 得到该学期最终绩点后，再并入个人累计 GPA

累计规则：

```text
个人GPA = 所有已完成学期最终绩点的平均值
```

## 汇报策略

汇报策略在基础图纸评级生成后结算，不改变 `进度` 和 `作品质量`，只影响 `最终图纸评级 / 作业分 / 设计课绩点`。汇报成功率是策略固定值，不由属性动态计算；属性只决定策略是否可选。

评图界面不单独展示“汇报成功 / 汇报失败”的结果页，结算后直接进入最终图纸评级展示。玩家能感受到策略影响，但 UI 不把老师有没有被说服拆成另一场审判。

| 策略 | 内容简介 | 可选门槛 | 固定成功率 | 成功结算 | 失败结算 |
|---|---|---|---:|---|---|
| 我就是大师 | 把方案讲成宣言，把剖面讲成命运。老师如果吃这套，你就是大师；不吃这套，你就是在给自己加戏。 | 设计水平 >= `55`，创意审美 >= `50`，汇报表达 >= `45` | `55%` | 最终图纸评级上升 `1` 档，作业分 +4，压力 +3 | 作业分 -3，压力 +6 |
| 高级技术流 | 参数、节点、结构逻辑一顿输出，听起来像能建出来，也可能像软件教程现场。 | 软件技术 >= `55`，设计水平 >= `50`，汇报表达 >= `40` | `60%` | 最终图纸评级上升 `1` 档，作业分 +3，压力 +2 | 作业分 -2，压力 +4 |
| 求求你别挂我 | 不装了，摊牌了，孩子真的改不完了。尘埃还未落定，但老师的慈悲也不是无限资源。 | 无门槛；基础图纸评级为 `F` 时也可选择 | `25%` | 若基础图纸评级为 `F`，最终图纸评级改为 `D`；若基础图纸评级为 `D`，作业分 +2；其他评级作业分 +1；压力 +8，精力 -4 | 维持基础图纸评级，作业分 -2，压力 +10，精力 -6 |
| 直接念 PPT | 逐字朗读，稳定发挥，稳定尴尬。至少不会突然把老师讲醒，也不会把自己讲飞。 | 无门槛 | `100%` | 不升档；若基础图纸评级为 `A / B / C`，最终图纸评级下降 `1` 档；若基础图纸评级为 `D / F`，不再降档；作业分 -4 | 同成功结算 |

汇报策略通用规则：

- `A` 已是最高评级，成功升档时不再升档，改为作业分 +5。
- `求求你别挂我` 是唯一允许在基础图纸评级为 `F` 时尝试补救的策略；若进度未达硬门槛或质量低于 F 门槛，仍可尝试，但成功率固定较低且代价较高。
- 除 `直接念 PPT` 外，其余策略失败时不直接降档，只扣作业分和状态。
- 汇报策略造成的最终评级变化会影响本学期设计课绩点；作业分用于后续竞赛、作品集和路线门槛。
- 汇报策略结算导致精力小于 `0` 时，立即触发 `被迫停学`。

导师偏好原则：

- 每位导师只有 `1` 个明显偏好策略
- 每位导师只有 `1` 个明显厌恶策略
- 其余策略视为中性
- 导师偏好不改变策略固定成功率，只影响作业分与状态。
- 偏好策略成功时额外作业分 +2；厌恶策略无论成功失败都额外压力 +2。

## 体制内路线组

体制内路线必须先锁定 `选调 / 考公 / 考编` 之一，三者互斥。最终除属性门槛外，还必须完成行测考试。

### 行测考试

- 题库总量：`30` 道单选题
- 每次抽取：`10` 道
- 难度组成：`6` 道简单题 + `4` 道困难题
- 内部先记录答对题数，再折算为现实分数线
- 答题过程中不反馈单题对错，也不显示正确答案
- 大五上评图结束后，若已锁定 `选调 / 考公 / 考编` 且达到对应报名门槛，则强制进入行测考试阶段
- 行测结束后只记录内部得分与是否过线，不向玩家公开分数、答对题数或上岸结果
- 选调、考公、考编是否成功统一延后到大五下毕业设计答辩后的结局结算再揭示
- 折算规则：答对 `10 / 9 / 8 / 7 / 6` 题时，分别折算为 `150 / 145 / 140 / 135 / 130` 分；答对 `5` 题及以下统一视为 `130` 分以下

行测分数线：

| 方向 | 档位 | 分数线 |
|---|---|---:|
| 选调 | 生源地选调生 | `145` 分以上（至少答对 `9` 题） |
| 考公 | 国家部委层 | `145` 分以上（至少答对 `9` 题） |
| 考公 | 省市厅局层 | `140` 分以上（至少答对 `8` 题） |
| 考编 | 教师岗 | `140` 分以上（至少答对 `8` 题） |
| 考编 | 事业单位综合岗 | `135` 分以上（至少答对 `7` 题） |
| 考编 | 行政管理岗 | `130` 分以上（至少答对 `6` 题） |

失败回落：

- 锁定选调但未满足门槛或行测线，读取 `回乡待招录`。
- 锁定考公或考编，若满足 `汇报表达 >= 58`、`人际交往 >= 60`、`抗压能力 >= 58`，且行测达到 `130` 分以上（至少答对 `6` 题），但未满足更高档位，则读取 `基层服务`。
- 锁定考公或考编但未满足对应门槛或未达到 `基层服务` 条件，读取 `继续备考`。
- 体制内方向失败后不跳转建筑工作、转行或升学成功结局。

## 毕业设计完成门槛

顺利毕业必须完成毕业设计；否则无论路线是否达标，都读取 `没毕业`。

大五上和大五下共同承载同一个毕业设计，进度与作品质量不在大五上结束后清零。大五毕业设计的进度与作品质量上限均为 `200`。

大五上期中汇报条件：

- 大五上毕业设计期中汇报前，毕业设计进度必须达到 `100 / 200`；未达成则本次评图直接为 `F`
- 大五上毕业设计期中汇报的作品质量按梯度判定，至少达到不被评为 `F` 的门槛

毕业设计完成条件：

- 大五下毕业设计评图等级为 `D` 或以上
- 大五下评图前毕业设计进度必须达到 `200 / 200`
- 大五下毕业设计作品质量折算后的质量评图分必须达到 `D` 或以上
- 大五下毕业设计答辩一结束，立即进入结局系统结算；此前所有路线考试、申请、投递的内部结果都在此时统一读取

状态收尾：

| 状态收尾 | 条件 |
|---|---|
| 昂扬 | 通关时精力 >= `60` 且压力 <= `50` |
| 复杂 | 通关时不满足昂扬，也不满足沉重 |
| 沉重 | 通关时精力 < `35` 或压力 > `75` |

## 强路线稀有度

强路线不是失败条件，只影响成功通关时的主结局优先级。

| 强度 | 路线 | 目标体感 |
|---|---|---|
| 顶级强路线 | 保研直升、顶级名校留学、选调回乡、大师建筑事务所 | 需要明确规划，普通游玩不稳定达成 |
| 中强路线 | 考研上岸、普通留学、国企设计院、AI 新职业 | 合理规划可达成 |
| 普通路线 | 地方设计院、普通事务所、非 AI 转行、状态毕业 | 大多数未失败玩家可回落 |

调参原则：

- 若强路线过多触发，优先上调 `作品集 / GPA / 实习价值 / AI 相关经历` 门槛
- 若普通通关过难，优先降低失败压力，不降低强路线门槛
- `稳定毕业 / 昂扬毕业 / 带伤毕业` 不读取作品集、竞赛、实习等强路线条件

## 中途失败阈值

- 生活费断裂：任意结算后金钱小于 `0`，读取人生结局 `你破产了`
- 压力失控：压力 `> 90` 连续 `2` 周
- 连续挂科退学：连续 `2` 次评图 `F`
- 被迫停学：任意结算后精力小于 `0`，立即读取人生结局 `被迫停学`
