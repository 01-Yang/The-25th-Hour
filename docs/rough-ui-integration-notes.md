# 粗 UI 与游戏逻辑结合记录

最后更新时间：`2026-06-01 +08:00`

## 当前粗 UI 目标

粗 UI 不是最终视觉稿，而是规则和交互流程的可玩验证层。当前版本优先保证这些内容不丢：

- 开头加载态。
- 开始界面完整入口。
- 开始新游戏后才输入玩家名字、大学名字和随机种子。
- 角色抽取在正式开始第一学年前发生。
- 角色抽取完成后，先抽取第一学年导师。
- 固定开学流程、选课、周行动、随机事件、模型周、课程题、评图、暑期事件、结局都经由命令层推进。
- 设置入口在游戏中保留，并包含后续精 UI 需要的系统入口。
- 音乐播放器和 LRC 歌词测试入口先接入粗 UI。
- 日志展示每次结算的数值变化。

- 你的名字
- 大学名字
- 随机种子

- 玩家排行榜
- 结局与成就
- 放弃学业重新开始
- 作者的话
- 请作者喝咖啡续命

- 保存

## 周流程

当前周流程为：

```text
进入一周
-> 玩家执行本周行动
-> 行动次数用完后自动进入周结算
-> 触发本周随机事件
-> 随机事件全部确认后自动进入下一周或学期末流程
```

第一学期第一周的随机事件会强制在第一周行动结束后触发。普通随机事件在后续周行动结束后按概率触发。

## 弹窗规则

当前粗 UI 弹窗约定：

- 随机事件没有交互选项时，按钮文案直接显示本次数值结算。
- 有交互选项的事件，第一层按钮只显示选项。
- 玩家选择后，下一层弹窗显示说明和数值结算。
- 第二层弹窗按钮统一为 `确定`。
- 固定流程和暑期事件也使用“先选项，后说明与数值结算”的粗 UI 表达。

## 单屏原则

桌面端粗 UI 以 `100dvh` 单屏为目标：

- 左侧为档案、音乐/LRC、设置。
- 右侧为状态、行动、日志和评图记录。
- 行动和日志在局部容器内压缩展示，避免玩家为了主流程下滑页面。

移动端会保留可访问性和基本可用性，必要时允许页面自然展开；最终精 UI 阶段再单独做移动布局。

## 随机种子用途

随机种子是整局游戏的可复现随机源。当前会影响：

- 初始角色三选一。
- 重抽角色。
- 导师候选抽取。
- 随机事件触发判定。
- 随机事件抽取。
- 课程题抽题。
- 评图汇报策略的成功判定。

同一个种子、同一套选择路径，应得到同样的抽取和判定结果。这对调试、平衡数值和复现玩家反馈很重要。

## 后续精 UI 注意点

- UI 不直接改 `GameState`，所有推进继续走命令层。
- 随机事件应继续在周行动结束后出现，不回到开周抢先弹出。
- 设置入口和开始界面入口不应在视觉升级时被删。
- 音乐和 LRC 可以换成正式资源清单，但不要参与规则判定。
- 排行榜和成就当前仍是入口占位，后续需要接正式存储和 API。
