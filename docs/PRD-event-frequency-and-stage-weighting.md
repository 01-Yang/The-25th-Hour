# 随机事件频率与阶段权重 PRD

状态：已收敛。现行规则已同步到 `systems.md / numbers.md / events.md`；若本文与核心文档冲突，以核心文档为准。

## Problem Statement

本专题发起前，单局随机事件体系的目标触发次数仍按旧口径 `84` 次校准，大一、大二阶段的事件密度偏保守，早期校园生活、社交互动、学业压力和专业成长的存在感不够稳定。现已确认把单局目标事件触发次数提高到 `90-95` 次，并让大一、大二更热闹，但不能让早期体验变成纯社交或纯娱乐；学业交互也必须保持足够权重。

同时，AI 相关语境需要从大一下开始进入游戏，而不是等到大二上。

## Solution

将单局事件目标调整为 `90-95` 次，标准目标按 `92` 次校准。`92` 次事件发生在 `60` 个正式周里，固定流程事件与暑假写生节点不计入事件目标，普通随机事件、交互事件、模型周事件和万里路事件计入目标。

大一、大二作为高触发期，目标各 `22` 次；大三 `18` 次，大四 `16` 次，大五 `14` 次。普通随机事件与交互事件在体感上经常前后呼应，形成“日常事件 + 交互事件”的节奏；连续 `2` 次普通随机事件保留为 `15%` 的小概率。高触发阶段若某周自然抽取后没有事件，则周末保底触发 `1` 次普通随机事件或交互事件。

大一、大二交互事件权重上调时，按学业交互、社交/朋友/暧昧、娱乐/恢复三类做平衡，推荐结构为 `40% / 35% / 25%`。

AI 相关事件从大一下开始开放，但仍作为随机事件体系的语境标签，不新增独立 AI 事件池。

## User Stories

1. As a player, I want大一、大二有更高事件密度, so that early campus life feels alive.
2. As a player, I want交互事件后经常跟着一个日常随机事件, so that choices feel connected to the week rather than isolated.
3. As a player, I want偶尔连续遇到普通随机事件, so that daily life can have small streaks and surprises.
4. As a player, I want早期社交、朋友、暧昧、娱乐事件更多, so that入学阶段有真实校园感.
5. As a player, I want早期学业交互也保持高权重, so that建筑学压力和专业成长不会被生活事件淹没.
6. As a player, I want大五事件密度略降, so that毕业设计和结局节奏不被频繁打断.
7. As a player, I wantAI 从大一下开始出现, so that时代变化更早进入建院生活.
8. As a designer, I want明确的单局目标次数, so that后续模拟验证有标准。
9. As a designer, I want按学年统计事件次数, so that阶段体验可以被调参。
10. As a designer, I want按事件池统计目标次数, so that普通随机事件、交互事件、模型周事件和万里路事件比例不会跑偏。
11. As a developer, I want固定流程事件不计入事件目标, so that开学典礼、军训、毕业典礼不会挤占随机事件空间。
12. As a developer, I want交互事件和普通随机事件的联动规则清楚, so that实现抽取器时不需要猜节奏。
13. As a developer, I want连续普通随机事件概率有目标值, so that可以用模拟跑局验证。
14. As a developer, I wantAI 仍不单独开池, so that事件体系结构不膨胀。
15. As a content designer, I want大一、大二交互分类占比明确, so that新增事件时知道该补学业、社交还是娱乐。

## Implementation Decisions

- 单局事件标准目标为 `92` 次，允许自然波动到 `90-95` 次。
- 固定流程事件、暑假写生节点不计入事件目标。
- 学年目标为：大一 `22`、大二 `22`、大三 `18`、大四 `16`、大五 `14`。
- 事件池目标为：普通随机事件 `46`、交互事件 `34`、模型周事件 `6`、万里路事件 `6`。
- 普通随机事件与交互事件在体感上经常前后呼应，但不按固定比例硬切。
- 高触发阶段若某周自然抽取后没有事件，则周末保底触发 `1` 次普通随机事件或交互事件。
- 连续 `2` 次普通随机事件的目标概率为 `15%`。
- 连续 `2` 次交互事件不是目标节奏，除保底或阶段限定外应降权。
- 大一、大二交互事件推荐权重为：学业交互 `40%`、社交/朋友/暧昧 `35%`、娱乐/恢复 `25%`。
- AI 相关事件从大一下开放，仍通过 `ai_related` 与 `ai_experience_delta` 标记，不新增独立事件池。

## Testing Decisions

- 通过模拟跑局验证单局事件次数是否稳定落在 `90-95` 次。
- 验证固定流程事件和暑假写生节点不会计入事件目标。
- 验证大一、大二平均事件密度高于大三、大四、大五。
- 验证普通随机事件、交互事件、模型周事件、万里路事件的目标次数接近 `46 / 34 / 6 / 6`。
- 验证普通随机事件与交互事件在体感上经常前后呼应。
- 验证连续普通随机事件概率接近 `15%`。
- 验证大一、大二交互事件中学业类不会低于目标结构，避免早期纯社交化。
- 验证 AI 相关事件最早可在大一下触发，且大一下至大三下总保底仍为 `4` 次。

## Out of Scope

- 不新增独立 AI 主系统。
- 不新增第三个交互选项。
- 不重写既有事件正文。
- 不改变结局系统。
- 不把固定流程事件纳入事件次数目标。

## Further Notes
