# 跑局模拟到内容设计对接文档

最后更新时间：`2026-05-26 +08:00`

本文用于把当前 headless simulator 已验证的跑局结果，直接转成内容设计工作。它不是新的规则源；流程规则仍以 `systems.md` 为准，数值仍以 `numbers.md` 与 `simulator/balance.ts` 的阶段性实现为准，具体正文仍回到 `content-plan.md`、`events.md`、`endings.md` 和 `question-banks.md` 维护。

## 当前模拟结论

当前模拟器已经跑通以下核心闭环：

- 固定 seed 普通局可以完整跑满 `60` 周，并稳定进入普通毕业结局。
- 四类中途失败已经可复现：生活费断裂、被迫停学、压力失控、连续评图失败。
- 事件启用后，同 seed 事件序列可复现；模型周事件只出现在每学期第 `5` 周；AI 相关事件只在大一后半到大三阶段出现。
- 后期路线流程已验证为：大四上出现路线意向，大五上形成正式路线，大五上评图后缓存隐藏路线结果，大五下完成毕业设计后再由结局解析器统一揭示。
- 单局报告已经能输出隐藏路线判定时的 GPA、作品集、六项属性、考试或申请分数、失败或未达最高档原因。
- 批量报告已经能输出竞赛奖项分档、竞赛奖金、实习最终档位分布、路线诊断计数。

这说明内容设计现在可以从“系统能不能跑通”进入“玩家每个节点看到什么、为什么相信这个结果、失败时怎么理解”的阶段。

## 内容设计总目标

当前内容设计要服务三个目的：

1. 让玩家读懂五年建筑生涯的节奏。

玩家应该能感受到大一到大五的压力递进：从适应建院、学会画图、第一次评图，到竞赛、实习、后期路线、毕业设计和最终分岔。

2. 让每个结果看起来是自己一路选择的结果。

结局不能像系统突然判死。每条路线都需要前置提醒、过程反馈、等待结果、最终揭示，让玩家能回想起自己哪些行为推动了这个结果。

3. 让失败有解释，不只是一句失败。

模拟器已经能给出失败诊断，例如 GPA 不足、表达不足、考试分数不足、留学概率失败。内容设计要把这些诊断翻译成玩家能理解的文本。

## 已可直接对接的内容节点

### 普通毕业

模拟状态：

- `normal` 和 `balanced` 都能稳定完整读完 `60` 周。
- 未参与或未读取成功路线时，最终走状态毕业。
- 通关时根据最终状态读普通或沉重收尾。

内容设计要点：

- 普通毕业不应该写成失败，而应该写成“大多数人的真实出口”。
- 文案重点是五年经历的回望、未定人生的开放感、建筑学训练留下的痕迹。
- 沉重毕业需要体现身体和精神损耗，但仍然承认玩家走到了终点。

需要检查的内容：

- `endings.md` 中 `稳定毕业——平平淡淡才是真！`
- `endings.md` 中 `带伤毕业——看起来有点糟`
- 毕业典礼固定事件正文
- 结束曲出现时机与情绪衔接

### 四类中途失败

模拟状态：

- 金钱小于 `0` 触发生活费断裂。
- 精力小于 `0` 触发被迫停学。
- 压力大于 `90` 连续 `2` 周触发压力失控。
- 连续 `2` 次评图为 `F` 触发连续评图失败。

内容设计要点：

- 四类失败都需要有提前预警文本，不要只在终点突然宣判。
- 失败正文要强调“这个系统在惩罚什么”：金钱惩罚资源断裂，精力惩罚过度透支，压力惩罚长期崩溃，评图惩罚课题连续失守。
- 中途失败不是结尾动画的重点，而是玩家复盘和下局调整的入口。

需要补齐的内容：

- 首次金钱高危提示。
- 首次精力高危提示。
- 首次压力高危提示。
- 首次评图 `F` 后的补救提醒。
- 连续失败前的第二次风险提醒。
- 失败后复盘页的“主要原因”短句。

### 评图与 GPA

模拟状态：

- 每学期通过评图产生设计课绩点。
- GPA 会影响升学、留学、体制内路线。
- 普通局 GPA 均值约 `2.76`，平衡局约 `3.14`，后期路线策略会有更高或更偏科的表现。

内容设计要点：

- 评图文本要让玩家明白：作品进度和作品质量是两个维度。
- GPA 不是每周都要强调，但每次路线门槛受阻时要能解释。
- `D` 是过关，不是荣耀；`B` 以上才是玩家感到“不错”的节点。

需要补齐的内容：

- 每个评图等级的短版弹窗，用于高频展示。
- 每个评图等级的长版评语，用于详情页或日志。
- GPA 变化提示文案。
- 评图失败后的补救建议文案。

### 竞赛

模拟状态：

- 当前竞赛已记录三等奖、二等奖、一等奖和奖金。
- 分档现在是确定性表现分，不是最终概率表。
- 100 局事件基线中，不同策略的平均竞赛记录和奖项等级已经能拉开差异。

内容设计要点：

- 竞赛应表现为“作品走出专教”的节点，不直接增加作品集分数。
- 一等奖要稀有且有行业认可感。
- 三等奖不能写得太爽，它更像简历上终于多了一行。
- 未获奖也要有正文，因为投稿失败是重要学习反馈。

需要补齐的内容：

- 四个固定赛事的投稿入口文案。
- 每个赛事的未获奖、三等奖、二等奖、一等奖结果文案。
- 奖金入账提示。
- 简历写入提示。
- 首次投稿、首次获奖、累计获奖成就提示。

### 实习

模拟状态：

- 当前所有代表策略最终都能到达 `named_firm` 档位。
- 这说明实习 stub 目前更像属性成长记录器，不是难度来源。
- 模拟器已能记录实习档位升级：普通、强所、名企。

内容设计要点：

- 内容上应先准备三档实习文本，但不要把名企写成默认结果。
- 实习要承担“学生身份第一次被社会检验”的感觉。
- 普通实习也要有价值，不要只有名企才有叙事。

需要补齐的内容：

- 普通实习录取、入职、结束总结文案。
- 强所实习录取、入职、结束总结文案。
- 名企实习录取、入职、结束总结文案。
- 实习失败文案。
- 实习持续期间每周轻量事件或日志短句。
- 简历写入提示。

### 后期路线

模拟状态：

- 当前验证的是每个路线组的一个代表目标。
- `postgrad` 当前代表普通考研上岸。
- `overseas` 保留申请概率失败。
- `civil_service` 有成功、兜底、失败三档。
- `architecture_job` 当前代表地方设计院，成功率很高。
- `career_change` 当前代表新媒体内容，少量因表达不足失败。

内容设计要点：

- 大四上路线意向是“提醒和想象”，不能像最终选择。
- 大五上正式路线是“承诺”，要有确认感。
- 结果在毕业时揭示，所以路线过程要有等待感和悬念。
- 每条路线都需要“成功文本、失败或回落文本、等待结果文本、未达条件解释文本”。

需要补齐的内容：

- 大四上路线提醒后的各路线说明。
- 大五上正式参与确认文案。
- 路线等待结果文案。
- 路线未进入考试、申请或投递条件时的内部解释文本。
- 最终结局揭示前的过渡文案。

## 路线内容设计切片

以下切片按 `$to-issues` 的垂直切片思路整理：每一条都应该能独立完成一条从“玩家选择/状态”到“内容反馈/结局解释”的完整体验。

### 1. 普通毕业体验闭环

类型：AFK

依赖：无

要完成：

- 普通毕业、带伤毕业的结局前过渡文案。
- 毕业典礼到结局揭示的情绪衔接。
- 未参与路线、路线未成功、普通状态毕业三种语境下的短句差异。

验收标准：

- 玩家完整读完五年但没有强路线成功时，不会感觉被系统判成失败。
- 普通毕业和带伤毕业能明显区分最终状态。
- 文案不引入新的规则条件。

### 2. 中途失败预警与复盘闭环

类型：AFK

依赖：无

要完成：

- 金钱、精力、压力、连续评图失败四类预警文案。
- 四类失败结局前的最后一击文案。
- 失败复盘页原因短句。

验收标准：

- 每类失败至少有一个提前预警和一个最终解释。
- 玩家能知道下一局应该改什么。
- 文案与现有失败结局标题一致。

### 3. 竞赛投稿与结果闭环

类型：AFK

依赖：普通评图和作品集内容已稳定

要完成：

- 四个固定赛事的投稿介绍。
- 未获奖、三等奖、二等奖、一等奖结果文本。
- 奖金、简历写入、成就触发提示。

验收标准：

- 每次竞赛结果都能展示奖项、奖金、简历写入状态。
- 未获奖不是空反馈。
- 一等奖明显比二等奖更稀有、更有分量。

### 4. 实习申请与结束闭环

类型：AFK

依赖：实习数值最终表待定，但内容可先按三档写

要完成：

- 普通、强所、名企三档录取文案。
- 申请失败文案。
- 三周实习期间日志短句。
- 实习结束总结文案。

验收标准：

- 普通实习也能让玩家觉得有收获。
- 名企实习有明显门槛感。
- 实习结束能总结工资、成长和简历写入。

### 5. 考研路线闭环

类型：HITL

依赖：需要确认保研与考研是否拆开首批实现

要完成：

- 大四想考研时的路线说明。
- 大五正式报名确认。
- 考试后等待结果文案。
- 普通上岸、强校上岸、再考一年等结果衔接。
- 未达门槛诊断文案：GPA 不足、作品集不足、设计/软件/抗压不足、考试分不足。

验收标准：

- 玩家能区分“努力准备中”和“正式报名后不能改”的阶段。
- 考试结果不提前公布，但等待反馈存在。
- 失败原因能被翻译成玩家看得懂的句子。

### 6. 留学路线闭环

类型：AFK

依赖：留学院校档位内容已存在

要完成：

- 留学申请前说明。
- S/A/B/C 档申请语气差异。
- 等 offer 的等待文案。
- 申请失败文案，尤其处理概率失败而非数值不足的情况。

验收标准：

- 玩家能感受到留学路线是作品集、GPA 和不确定性共同作用。
- 概率失败不能写成玩家完全没努力。
- 不同院校档位有不同情绪重量。

### 7. 体制内路线闭环

类型：HITL

依赖：需要确认考公、考编、选调首批是否共用一个玩法入口

要完成：

- 考公、考编、选调三类路线说明。
- 行测考试后的等待文案。
- 成功、兜底、失败三档结果过渡。
- 诊断文案：GPA 不足、表达不足、考试分不足。

验收标准：

- 三条体制内路线不能互相串线。
- 兜底结果不是失败文本的复制。
- 玩家能理解表达、人际、抗压为什么影响这条路。

### 8. 建筑就业路线闭环

类型：HITL

依赖：需要先确认首批岗位目标，当前模拟只代表地方设计院

要完成：

- 地方设计院、国企设计院、外企事务所、大师事务所等岗位投递说明。
- 简历只能投一家的确认文案。
- 成功和求职待定结果过渡。
- 实习、作品集、软件、设计门槛不足时的诊断文案。

验收标准：

- 不同岗位的野心和风险明显不同。
- 地方设计院可以作为稳定就业，不应写成低价值失败。
- 大师事务所与外企需要高门槛感。

### 9. 转行路线闭环

类型：HITL

依赖：需要确认首批转行岗位，当前模拟只代表新媒体内容

要完成：

- AI 产品经理、游戏场景建模师、销售/商务、新媒体内容、插画师的路线说明。
- 简历投递确认。
- 成功和转行失败结果过渡。
- 诊断文案：表达不足、审美不足、软件不足、作品集不足。

验收标准：

- 转行不是背叛建筑，而是建筑训练被带到别处。
- 不同岗位读取不同能力，不要都写成“逃离画图”。
- 失败文本要承认尝试本身的重量。

## 诊断文案对照表

这些是模拟器当前会输出或即将继续扩展的诊断项，内容设计应为每项准备玩家可见短句。

| 诊断项 | 玩家可见解释方向 |
|---|---|
| `gpa_below_threshold` | 你的课程与评图成绩还没有支撑这条路线。 |
| `portfolio_below_threshold` | 你的作品集厚度还不够，缺少能打动对方的代表作。 |
| `design_below_threshold` | 方案能力还差一口气，设计判断没有稳定到目标要求。 |
| `software_below_threshold` | 软件和制图能力不足，难以支撑更高强度的输出。 |
| `aesthetic_below_threshold` | 图面和审美表达还不够成熟。 |
| `presentation_below_threshold` | 你还没有把自己的想法讲清楚。 |
| `social_below_threshold` | 这条路需要更多沟通、协调和人际处理能力。 |
| `resilience_below_threshold` | 你的抗压能力还不足以撑过这条路线的筛选。 |
| `internship_below_threshold` | 简历里缺少足够有分量的实习经历。 |
| `recent_failed_reviews_above_threshold` | 最近几次评图记录影响了对方对稳定性的判断。 |
| `exam_score_below_threshold` | 考试表现没有达到分数线。 |
| `overseas_probability_roll_failed` | 你达到了基本条件，但申请仍然有不确定性，这次没有等到好消息。 |

## 内容优先级

建议内容设计按以下顺序推进：

1. 先补玩家高频看到的内容：评图、普通毕业、中途失败预警。
2. 再补能直接解释模拟结果的内容：路线等待、路线诊断、竞赛结果、实习总结。
3. 最后扩展强路线和多目标岗位：强校、强留学、选调、国企、外企、大师事务所、多个转行岗位。

原因是当前模拟器已经证明普通局和代表路线能跑通，但强路线和多岗位还只是内容池与规则文档中的目标，并没有全部进入可执行模拟。

## 不要现在做的事

- 不要把所有路线结局一次性重写。先补结果揭示和诊断短句，再逐步扩写长结局。
- 不要把竞赛奖金写成主要成长来源。竞赛的主价值是简历和身份确认。
- 不要把实习默认写成名企成功。当前模拟全员名企只是 stub 过松，不是最终内容口径。
- 不要把普通毕业写成低价值坏结局。它是基准人生出口。
- 不要新增隐藏属性解释内容。所有文案都应围绕现有六项属性、GPA、作品集、金钱、精力、压力、竞赛、实习和路线。
