# 模拟结论回写文档

最后更新时间：`2026-05-26 +08:00`

## 这份文档是干什么的

这份文档只做一件事：把这一轮跑局模拟里已经稳定、可以长期保留的内容结论，整理成可并回主线的文档口径。

它不是：

- 不是模拟器使用说明
- 不是临时调参记录
- 不是代码实现文档
- 不是细模块文案草稿堆放处

它要回答的是：这一轮模拟之后，主线文档到底应该稳定成什么样。

## 本轮可并回主线的结论

### 1. 竞赛口径收口

竞赛玩家可见层不再保留过细拆分，统一收口为：

- 竞赛结果弹窗只保留 `未获奖 / 三等奖 / 二等奖 / 一等奖`
- 奖金提示统一为：`竞赛奖金 ￥{amount}`
- 删除“竞赛简历写入提示”
- 删除“竞赛成就提示”

需要保留细分的只有规则层信息，例如具名赛事、投稿门槛、概率分档、奖金倍率；这些继续留在规则与数值文档，不再进入玩家高频可见文案层。

### 2. 实习口径收口

实习玩家可见层不再保留过细阶段提示，统一收口为：

- 成功：`你收到了一封实习录取邮件。`
- 失败：`你盯着那封拒绝信看了很久。`

同时：

- 删除其他“入职”“结束总结”“额外提示”类文案口径
- 保留实习期间三周短事件
- 每条短事件保留对应的轻量属性加成

这意味着实习的玩家体验结构现在很明确：`申请结果 -> 三周过程短事件 -> 路线读取其价值`。

### 3. 实习价值改为累计口径

建筑就业路线读取实习时，不再用“拿过的最高档实习”来理解，而改为累计 `实习价值`：

- 普通实习：`+1`
- 强事务所实习：`+2`
- 名企实习：`+3`

这里的含义是：实习价值反映的是累计履历厚度，而不是一次最高光经历。

### 4. 建筑就业相关文档同步累计颗粒度

与实习相关的主线文档，统一按累计 `实习价值` 写条件，不再混用“任意实习 / 名企实习 / 最高档实习”这种旧口径。

当前主线文档应统一成这类表达：

- `实习价值 >= 1`
- `实习价值 >= 2`
- `实习价值 >= 3`

### 5. 名企不是默认叙事中心

这一轮模拟已经说明：名企实习不是默认结果。

因此主线内容必须承认：

- 普通实习需要正式文案重量
- 强所实习需要正式文案重量
- 拒信也要成为正式反馈的一部分

不能把实习系统写成“只有名企成功才算有内容”。

### 6. 竞赛奖金不是主要成长来源

竞赛奖金可以存在，但它不是竞赛内容的主价值。

竞赛更重要的作用是：

- 给玩家一个“作品走出专教”的节点
- 提供履历确认感
- 提供路线读取时可理解的背景
- 提供身份成长感

所以主线文案不应把竞赛写成刷钱系统。

## 本轮已经同步到哪些主线文档

本轮沉淀对应的是下面这些文档：

- `docs/content-plan.md`
- `docs/events.md`
- `docs/numbers.md`
- `docs/endings.md`

其中各自承担的职责应保持稳定：

| 文档 | 承担内容 |
|---|---|
| `content-plan.md` | 玩家可见提示、结果弹窗、阶段提示 |
| `events.md` | 实习期间短事件与短文本颗粒度 |
| `numbers.md` | 门槛、收益、概率、实习价值规则 |
| `endings.md` | 路线结局与结局条件口径 |

## 不并回主线的内容

下面这些不属于这次并回主线的范围：

- 模拟脚本本身
- 批量跑局工具
- 临时统计报表
- 调参过程记录
- 已经被删掉的细模块文案口径

这次要并回去的是“模拟后的稳定结论”，不是“模拟过程”。

## 这份文档怎么用

你后面用它，按下面这个顺序就够了：

1. 先把这份文档当成“本轮结论清单”。
2. 检查 `content-plan.md / events.md / numbers.md / endings.md` 是否都已经按这里的口径统一。
3. 发一个 docs-only PR，只提交这几份文档。
4. 合并后，把 `main` 作为后续内容设计和实现对齐的统一依据。

简单说，这份文档不是给玩家看的，也不是给程序直接读取的；它是给你和后续协作者看的“这轮模拟到底沉淀了什么”。

## 并回 main 的目的是什么

并回 `main` 的目的，不是为了把模拟代码带回去，而是为了把已经确认的内容口径沉淀到主线。

它的价值有三点：

1. 让主线文档停止继续引用那些已经被你删掉的细模块口径。
2. 让后续写正文、做实现、补事件的人都以同一套颗粒度为准。
3. 把“实验分支里的阶段性结论”变成“主线认可的正式结论”。

你可以把它理解成：

- 模拟分支负责试错
- docs-only PR 负责沉淀结论
- `main` 负责保存当前认可版本

## 当前需要你记住的一件事

如果这次只做 docs-only PR，那么并回 `main` 的是“文档口径”，不是“模拟器实现”。

也就是说：

- 主线文档会先统一为累计 `实习价值`
- 但模拟器代码如果还没改，就还不是这个实现口径

这不影响你先合并这次文档 PR；只是后面如果要做“文档与代码完全一致”，还需要再补一次实现对齐。
