# 评图、万里路与实习收敛 PRD

状态：已收敛。现行规则已同步到 `systems.md / numbers.md / events.md / endings.md`；若本文与核心文档冲突，以核心文档为准。

## Problem Statement

当前架构中，评图系统曾同时存在“综合总分公式”和“进度/质量硬门槛”的两种口径，容易导致玩家明明进度没做完却靠汇报或导师修正过关，也会让后续程序实现难以判断 `F` 的来源。现行规则改为三段：硬门槛、基础图纸评级、汇报策略修正最终评级。

同时，建筑生的万里路已经从整周占用改为消耗 `2` 次行动机会，但旧文档仍残留开放年级、模型周/赶图周限制和事件上限口径。实习系统也需要从固定录取改为概率录取，并降低早期门槛与持续消耗，避免大一开放后无法参与，或中后期属性过快满级。

## Solution

评图改为三段制：先判定进度与质量硬门槛，再由作品质量决定基础图纸评级和基础作业分，最后由汇报策略修正最终图纸评级、作业分和设计课绩点。大一至大四普通课题的进度上限为 `100`，评图前进度未满 `100` 时基础图纸评级为 `F`；大五毕业设计的进度和作品质量上限为 `200`，大五上期中要求进度达到 `100`，大五下答辩要求最终进度达到 `200`。

建筑生的万里路从大二上开放，每次消耗 `2` 次行动机会，允许在模型周或赶图周使用。玩家选择前往当前地点后必定触发对应万里路事件，且该事件不计入普通正式周最多 `2` 次事件上限。第一次在模型周或赶图周使用万里路时，触发成长成就 `忙里偷闲`。

实习从大一开放，但申请不消耗行动次数，采用概率录取。普通实习、强事务所实习和名企实习都使用更低的最低门槛；刚满足门槛时概率较低，高度超过门槛时录取率达到 `100%`。实习一般持续 `3` 周，允许跨学期继承，已完成实习的单位不能再次申请，结束时展示总结并写入简历。

## User Stories

1. As a player, I want进度没做完时默认无法靠嘴皮子过关，但仍能用低概率高代价的 `求求你别挂我` 做最后补救, so that建筑学赶图压力更真实，也保留评图现场的戏剧性。
2. As a player, I want作品质量直接决定评图等级, so that我知道应该把时间投向方案质量。
3. As a player, I want大五毕业设计和普通学期区分数值上限, so that毕业设计有更长线的推进感。
4. As a player, I want万里路可以在模型周或赶图周使用, so that我能做出“忙里偷闲”的风险选择。
5. As a player, I want万里路事件必定触发且不挤掉普通事件, so that旅行投入有稳定反馈。
6. As a player, I want实习从大一就能看见入口, so that我能提前规划设计水平和软件技术。
7. As a player, I want实习申请有概率, so that刚够门槛时有忐忑，高度达标时有确定性。
8. As a player, I want实习不消耗行动次数, so that它更像履历申请而不是周行动替代品。
9. As a designer, I want汇报策略不再修改进度和作品质量，只影响最终图纸评级、作业分和设计课绩点, so that评图系统不会再出现双重公式。
10. As a designer, I want实习每周消耗更轻, so that工资收益和状态代价保持平衡。
11. As a developer, I want万里路是否计入周事件上限有明确规则, so that事件抽取器不会误吞必触发地点事件。
12. As a developer, I want实习跨学期继承规则明确, so that持续效果和总结页面可以稳定实现。

## Implementation Decisions

- 评图模块拆成硬门槛检查、基础图纸评级、汇报策略修正最终评级三步。
- 普通学期的 `进度 / 作品质量` 上限保持 `100`，大五毕业设计的 `进度 / 作品质量` 上限为 `200`。
- 大五毕业设计评图时，作品质量按 `floor(作品质量 / 2)` 折算为 `0-100` 的质量评图分。
- 普通学期中，约 `6-7` 次正常速度画图应能补满进度，约 `3-4` 次爆肝通宵赶图应能补满进度；事件进度加减允许让实际次数上下浮动。
- 汇报策略不改变 `进度` 或 `作品质量`，只在基础图纸评级后影响 `最终图纸评级 / 作业分 / 设计课绩点`；其中 `求求你别挂我` 是唯一允许在基础图纸评级为 `F` 时低概率补救为 `D` 的策略。
- 导师阶段任务、模型材料、事件和角色技能只改变评图前数值，不再产生独立评图总分。
- 万里路从大二上开放，允许模型周和赶图周使用。
- 万里路事件仍计入单局 `92` 次事件目标统计，但不受普通正式周最多 `2` 次事件上限限制。
- 实习申请不消耗行动次数，每学期最多申请 `1` 次，失败后当学期不可重投。
- 实习录取概率按设计水平和软件技术超过门槛的幅度提升，平均超额 `+20` 时达到 `100%`。
- 实习持续 `3` 周，跨学期继承，完成过的单位进入不可重复申请列表。

## Testing Decisions

- 模拟普通学期：进度为 `99`、作品质量为高分时，基础图纸评级为 `F`；只有选择并成功触发 `求求你别挂我` 时，最终图纸评级可补救为 `D`。
- 模拟普通学期：进度为 `100`、质量评图分分别落在 `44 / 45 / 60 / 75 / 85` 时，基础图纸评级应映射为 `F / D / C / B / A`。
- 模拟大五：大五上进度低于 `100` 时直接 `F`；大五下进度低于 `200` 时读取未完成毕业设计。
- 模拟模型周或赶图周使用万里路：应消耗 `2` 次行动机会、触发万里路事件、解锁 `忙里偷闲`，且不占用普通事件上限。
- 模拟实习申请：低于门槛为 `0%`，刚达门槛为 `35%`，高度达标为 `100%`。
- 模拟跨学期实习：剩余周数继承，结束时展示总结并写入简历。

## Out of Scope

- 不新增主行动。
- 不新增导师好感度或导师认可度。
- 不新增实习岗位池外的转行岗位。
- 不在本 PRD 内完成全局数值平衡；后续需要单独跑局模拟调整角色初始属性、行动收益、事件收益和路线门槛。

## Further Notes

本 PRD 收敛的是规则口径。具体数值仍以后续模拟为准，尤其是角色初始属性、实习门槛、行动收益、学期成长上限和路线触发率，需要进入专门的数值平衡轮次。
