# 2026-06-01 粗 UI 工作日志与问题记录

- 新增开头加载界面，展示指定加载文案。
- 重做开始界面为菜单入口，点击开始新游戏后才填写名字、大学和随机种子。
- 补齐开始界面入口：继续存档、作者的话、咖啡、深浅色、社区、排行榜、结局与成就、语言。
- 调整正式开局顺序：创建档案 -> 抽角色 -> 抽第一学年导师 -> 开学固定流程 -> 选课 -> 第一周。

- 改造事件弹窗：有选项时先只显示选项，选择后再显示说明和数值结算。

- 游戏中设置面板补齐入口：介绍与引导、排行榜、成就、重开、作者、咖啡、社区、语言、主题、保存。
- 加入粗 UI 音乐播放器，可手动选择 MP3 和 LRC 文件测试歌词同步。
- 日志增加数值变化显示。
- 视觉改为黑白灰复古像素风，辅以红色和深蓝高亮。
- 新增 `docs/rough-ui-integration-notes.md`，记录粗 UI 与游戏逻辑结合规则。

## 碰到的问题

- 当前 `web-app` 仍是临时试玩壳，还没有迁移到正式 `game-core / game-data` 包结构。
- 排行榜、结局与成就、社区、赞助、语言切换目前是入口占位，尚未接正式数据或外部接口。
- 音乐和 LRC 当前用本地文件选择测试，不保存资源清单，也不做歌词偏移校准。
- 单屏 UI 在桌面端可压缩展示，但移动端如果强行不滚动会牺牲可用性；当前移动端保留自然展开，后续精 UI 需要单独设计。
- 弹窗说明二段式已经覆盖固定事件、随机交互事件和暑期事件；项目选择、课程选择、导师选择仍保留说明预览，后续可按精 UI 再统一。
- 当前本地存档仍使用 `localStorage` 保存整局状态，正式首版应迁移到 `IndexedDB`。
- 视觉字体优先使用本机像素字体或等宽中文字体；如果用户机器没有对应字体，像素感主要由按钮、边框和布局呈现。
